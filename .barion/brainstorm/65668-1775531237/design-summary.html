편성관리 리디자인 — 최종 설계 요약

4개 화면 전체 설계. 확인 후 스펙 문서로 넘어갑니다.

1. 디자인 토큰 업데이트

Border Radius (네이버 기준)
4px — 뱃지, 태그
6px — 버튼, 인풋
10px — 카드
12px — 모달, 패널
컬러 적용 원칙 (Agentify 영감)
• CTA: 미묘한 그래디언트 + 소프트 섀도
• 표면: FAF9F7 → F2F0ED → white (3단계 레이어링)
• 보더: rgba 투명도 활용 (하드 라인 최소화)
• 상태 뱃지: border-left 3px 액센트 포인트
• 타임라인 바: 상태별 그래디언트 (한 톤 차이만)

2. 편성 목록 (list.html)

핵심 변경
목록 ↔ 타임라인 듀얼 뷰
  • 토글 스위치로 목록/타임라인 전환
  • 타임라인 = 미니맵 + 포커스 패턴
  • 오늘 마커 (빨간 수직선 + 날짜 라벨)
  • 미니맵에서 드래그로 포커스 구간 선택
타임라인 바 상태 표현
적용중
예약됨
일시정지
종료
정보 표시
  • 바 안: 편성명 + 기간
  • 우선순위: P1/P2 라벨
  • 동기화: 빨간/초록 점 인디케이터

3. 편성표 상세 (detail.html)

레이아웃 구조
┌─────────────────────────────┐
│ breadcrumb + 액션 버튼 │
├─────────────────────────────┤
│ 히어로: 제목 + 상태 │ 동기화 │
│ 미니 타임라인 진행바 │
├──────────────────┬──────────┤
│ 재생목록 구좌 │ 충돌알림 │
│ (카드 리스트) │ 동기화 │
│ │ 메타정보 │
└──────────────────┴──────────┘
핵심 변경
  • 히어로 블록: 제목+뱃지+동기화 경고를 한 줄에
  • 진행바: 30일 중 경과일 % + 오늘 마커
  • 구좌 테이블→카드: 썸네일+순번+상태
  • SSP 구좌: 파란 배경으로 시각 차별화
  • 삭제 구좌: dashed 보더 + 투명도
  • 사이드바: 충돌경고, 동기화이력, 메타정보

4. 편성표 생성 (new.html)

레이아웃
┌────────────────┬────────────┐
│ 폼 영역 │ 라이브 │
│ │ 프리뷰 │
│ - 편성표명 │ │
│ - 재생목록 선택 │ 미니 │
│ - 매체 선택 │ 타임라인 │
│ - 기간 설정 │ │
│ - 우선순위 │ 충돌 경고 │
│ │ (실시간) │
└────────────────┴────────────┘
핵심 변경
  • 2컬럼: 좌 폼 + 우 라이브 프리뷰
  • 프리뷰 타임라인: 매체/기간 입력하면 실시간 반영
  • 충돌 사전 감지: 기존 편성과 겹침 시 즉시 경고
  • 기존 편성 = 회색 바, 새 편성 = 초록 점선 바
  • 충돌 없으면 "충돌 없음 ✓" 표시

5. 충돌 해소 (conflict.html)

레이아웃
┌─────────────────────────────┐
│ 충돌 타임라인 시각화 │
│ (두 편성 겹침 구간 강조) │
├─────────────────────────────┤
│ 해소 옵션 A │
│ → 선택 시 타임라인 변화 │
│ 해소 옵션 B │
│ → 선택 시 타임라인 변화 │
│ 해소 옵션 C │
│ → 선택 시 타임라인 변화 │
└─────────────────────────────┘
핵심 변경
  • 상단 타임라인: 두 편성 바가 겹치는 구간을 빗금/투명도로 강조
  • 인터랙티브 프리뷰: 라디오 선택 시 타임라인이 before→after로 전환
  • 옵션 A: 우선순위 유지 → 바 겹침 그대로
  • 옵션 B: 기간 조정 → 바가 밀려남 애니메이션
  • 옵션 C: 취소 → 새 편성 바 사라짐
  • 겹침 구간에 경고색 배경 (#fef3e0)

6. 공통 시각 언어

타임라인 관통
목록(미니맵) → 상세(진행바) → 생성(프리뷰) → 충돌(비교) 모두 동일한 타임라인 시각 언어
오늘 마커 일관성
빨간 수직선 + 날짜 라벨. 목록·상세·생성·충돌 화면 모두 동일 패턴
상태 컬러 통일
적용중(그린) · 예약(블루) · 일시정지(옐로 dashed) · 종료(그레이) · 동기화실패(레드 dot)
