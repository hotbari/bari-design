A. 목록 + 모달

사용자 테이블 목록 위에 초대 모달이 떠오르는 방식. 입력 필드가 적을 때(이메일, 역할, 매체사) 적합. 화면 전환 없이 빠르게 처리.

적합한 경우: UC-01 어드민 초대 (단순 3필드)

B. 목록 + 슬라이드 패널

목록 오른쪽에서 패널이 밀려 나오는 방식. 초대 폼과 기존 목록을 동시에 볼 수 있어 중복 확인이 쉬움.

C. 멀티스텝 위저드 (모달)

이메일 → 역할 → 권한범위 3단계로 분리. 각 단계에서 오류를 즉시 감지. UC-03처럼 권한 범위 설정이 복잡한 경우 입력 부담 분산.

적합한 경우: 운영대행사 초대 (매체 선택 필요)

참고: 어드민 초대(UC-01)는 필드가 단순(이메일+역할+매체사)하고, 매체사 초대(UC-03)는 권한 범위 설정(매체 선택)이 추가됩니다. 두 흐름에 같은 UI 패턴을 쓸지, 다른 패턴을 쓸지도 결정 포인트입니다.
