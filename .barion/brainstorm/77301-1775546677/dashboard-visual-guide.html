3. Elevation 해석 — 3단계 깊이

원칙 4: "카드의 border를 전부 지워도 그룹핑이 읽히는가?"

바닥 (Level 0)
페이지 배경
bg-primary (neutral-50)
그림자 없음
표면 (Level 1)
KPI 카드, 알림 패널
bg-elevated (white)
border-default + shadow-sm
부유 (Level 2)
이상 감지 히어로 카드
error-50 배경
error border + 컬러 shadow

4. 스페이싱 해석 — 간격으로 ZONE을 구분

원칙 4: 관련 없는 요소 사이 의도적 빈 공간

--space-2 (8px) 카드 내부 요소 간 (라벨↔숫자, 뱃지 내부)
--space-3 (12px) 같은 ZONE 내 카드 간 간격
--space-6 (24px) ZONE 간 간격 — 시각적 호흡 (카드 내부의 3배)
--space-8 (32px) 메인 영역 좌우 패딩
핵심: 카드 내부(8px) → 카드 간(12px) → ZONE 간(24px) 으로 비율이 1 : 1.5 : 3. 기존 프로토타입은 전부 16px으로 동일했음.

요약: 토큰은 같고, 해석이 다르다

영역 기존 (공장형) 대시보드 해석
컬러 primary green 도처에 red=즉시, yellow=주의, gray=정상, green=행동유도만
숫자 크기 모든 stat-card 동일 28px 주인공 28px+ / 조연 22px / 배경 16px
Elevation 모두 shadow-sm + 1px border 3단계: 바닥 / 표면(sm) / 부유(컬러 shadow)
간격 전부 16px 8px → 12px → 24px (1:1.5:3 비율)
