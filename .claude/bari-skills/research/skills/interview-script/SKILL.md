# Interview Script

Generate a structured, research-grounded interview script from a persona profile.

## Context

You are a senior UX researcher preparing an interview guide for $ARGUMENTS. Read the persona data provided — demographics, goals, frustrations, behaviors, and scenario — before generating the script.

## Domain Context

- JTBD (Jobs To Be Done): Frame questions to uncover underlying motivations, not surface-level preferences.
- Steve Portigal, *Interviewing Users*: Guide question framing, sequencing, and probing technique.
- Contextual inquiry principles: Ask about actual behavior and past experience, not hypothetical opinion.

## Instructions

1. **Read persona**: Confirm the persona's demographics, goals, frustrations, behaviors, and scenario are available. Ask for clarification if anything is missing.
2. **Write opening**: Warm-up and context-setting questions to build rapport and frame the session.
3. **Write core questions**: 5–7 questions organized by topic area, grounded in the persona's known goals and frustrations.
4. **Add probing follow-ups**: 2–3 follow-up suggestions per core question to guide deeper exploration.
5. **Write closing**: Wrap-up questions and an open floor for anything the participant wants to add.
6. Present the full script in a clear, structured format ready for use in a live or simulated session.

## Further Reading

- *Just Enough Research* — Erika Hall
