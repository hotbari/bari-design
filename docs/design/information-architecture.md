## 4. 주요 태스크 플로우 (Top 5)

### Flow 1: 소재 업로드 → 검수 → 편성 (영업대행사 + 매체사 교차)

```
최유진(영업대행사)                    김서영(매체사)
    │                                    │
    ├─ S-14 규격 확인                     │
    ├─ S-12 소재 업로드 (UC-13)           │
    ├─ S-13 "검수중" 타임라인 확인         │
    │   ├─ 자동 검수 통과 (UC-14)         │
    │   │   └─ S-13 "검수완료" 확인       │
    │   └─ 자동 검수 실패 → 사유 복사      │
    │       └─ 광고주에 전달 → 재업로드    │
    │                                    ├─ S-13 수동검수대기 → 확인 (UC-15)
    │                                    ├─ S-19 재생목록에 소재 추가 (UC-21)
    │                                    └─ S-21 편성표 생성 → 즉시 반영 (UC-22)
    ├─ S-13 편성 연결 상태: "송출중" 확인   │
```

### Flow 2: 편성표 생성 → 충돌 해소 → 적용 → 확인 (운영대행사)

```
이준혁(운영대행사)
    │
    ├─ GNB 매체사 컨텍스트 스위처: 매체사 A 선택
    ├─ S-20 편성표 목록 → "열람 중 사용자 있음" 확인
    ├─ S-21 편성표 생성 (UC-22): 재생목록 선택, 매체/기간/우선순위
    │   └─ 충돌 감지 → S-22 충돌 해소 (UC-24): 우선순위 조정
    ├─ S-21 저장 → "예약됨"
    ├─ S-21 "즉시 반영" 클릭 → 변경사항 요약 확인
    └─ S-20 동기화 상태 배지: "동기화 완료" 확인
```

### Flow 3: 매체 장애 감지 → 대응 → 복구 확인 (어드민)

```
박진수(어드민)
    │
    ├─ S-26 알림 수신: "매체 오프라인" (인앱 + Slack)
    ├─ S-02 대시보드: 매체 헬스체크 이상 위젯 확인
    ├─ S-08 매체 상세: 헬스체크 이력 확인 → 현장 연락
    ├─ S-08 매체 복구 확인: 5단계 상태 "온라인-정상"
    └─ S-20 편성표: 일시정지 → 적용중 자동 복귀 확인
```

### Flow 4: 캠페인 등록 → 집행 → 완료 (매체사)

```
김서영(매체사)
    │
    ├─ S-16 캠페인 등록 (UC-17): Step1 기본 정보 → Step2 매체/기간 → Step3 예산
    ├─ S-17 캠페인 상세: "초안" 상태
    ├─ (시작일 도래) → 시스템 자동 "집행중" 전환 (UC-18)
    ├─ S-17 집행중 확인 → S-16 종료일/매체 수정 (UC-17a)
    └─ (종료일 경과) → 시스템 자동 "완료" (UC-19)
```

### Flow 5: 긴급 편성 발동 → 해제 → 복귀 확인 (어드민)

```
박진수(어드민)
    │
    ├─ S-24 긴급 편성 (UC-35): 매체 선택(전체), 소재 선택
    ├─ S-24 "긴급 송출 시작" → 확인 모달 → 발동
    │   └─ 모든 기존 편성/싱크 무시, 긴급 소재만 송출
    ├─ S-24 "긴급 송출 종료" → 해제
    ├─ S-24 복귀 상태: "복귀 완료 N/M대, 미복귀 K대"
    └─ S-20 종료일 경과 편성표 자동 종료 확인
```
