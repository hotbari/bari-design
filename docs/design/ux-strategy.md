# DOOH CMS — UX 전략

생성일: 2026-04-05
대상 페르소나: 박진수(어드민), 김서영(매체사), 이준혁(운영대행사), 최유진(영업대행사)

## 1. North Star Vision

**"CMS가 보여주는 상태와 현장 실제가 항상 일치하는 운영 도구"**

현재 DOOH 운영의 가장 큰 고통은 불확실성이다. CMS가 "적용중"이라고 표시해도 현장 전광판은 구버전 소재를 재생하고 있고, 검수는 언제 끝날지 알 수 없고, 편성 충돌이 생겼는지도 저장 후에야 안다. 이 시스템은 각 역할이 자신의 담당 범위 내에서 확신을 갖고 행동할 수 있는 운영 도구가 되어야 한다. 어드민 박진수는 장애를 30분 내에 파악·대응할 수 있어야 하고, 매체사 김서영은 광고주에게 "네, 정상 송출 중입니다"라고 자신 있게 답할 수 있어야 하며, 운영대행사 이준혁은 저장 후 안심감을 느낄 수 있어야 하고, 영업대행사 최유진은 광고주에게 정확한 온에어 일정을 약속할 수 있어야 한다.

## 2. 경쟁 환경 (Competitive Landscape)

| 일반 DOOH CMS 문제점 | 우리 CMS 차별화 |
|---------------------|----------------|
| 플레이어 상태를 온라인/오프라인 이진으로만 표시 | 5단계 동기화 상태 (온라인-정상 / 온라인-미완료 / 오프라인 / 이상감지 / 비활성) |
| 검수 결과를 기술 용어로 표시 ("codec mismatch") | 비기술적 언어 + 수치 명시 ("해상도 불일치: 1920×1080 필요, 1280×720 업로드됨") + 공유용 텍스트 복사 |
| 단일 사용자 UI — 역할 구분 없이 모든 메뉴 노출 | 역할별 완전 분리 대시보드 + 접근 범위 외 메뉴 완전 숨김 |
| 알림 단일 채널 의존 (Slack/인앱) | 채널 헬스체크 + 우선순위별 채널 분리 (긴급 → SMS/카카오, 일반 → 인앱) |
| "검수완료"가 "송출중"처럼 보이는 상태 표현 | 소재 상태 4단계 명시 (미편성 / 편성대기 / 송출중 / 송출종료) |
| 동시 편집 충돌을 저장 후에야 감지 | 사전 경고 ("지금 이 편성표를 열람 중인 사용자 있음") + 충돌 시 diff 패널 |
| 감사 이력 없음 — 분쟁 시 증거 부재 | 편성 수정 이력 (누가/언제/무엇) + 정산 리포트 연계 |

## 5. Design Brief 요약

### 무엇을 만드는가
DOOH 매체 운영의 편성·소재·알림·모니터링 전 과정을 역할별로 적정하게 노출하는 B2B 운영 도구. 네이버 SSP 연동 기반, 최대 수십 개 매체사·수백 대 매체 대상.

### 누구를 위한가
- **박진수 (어드민)**: 시스템 전체 상태를 한눈에 보고 장애를 30분 내 해소해야 하는 IT 운영자
- **김서영 (매체사)**: 다수 대행사 위탁 편성을 관리하며 광고주에게 현장 상태를 보고해야 하는 운영팀장
- **이준혁 (운영대행사)**: 3개 매체사·150대 매체를 동시에 실수 없이 처리해야 하는 주니어 매니저
- **최유진 (영업대행사)**: CMS에 비상주하며 소재 검수 결과와 송출 상태만 확인하면 되는 AE

### 성공의 기준
CMS 상태와 현장 실제가 일치하고, 불일치 시 해당 역할에게 즉시 도달하는 알림 체계가 작동한다.
각 역할이 자신의 담당 범위 내에서 다음 행동을 찾기 위해 동료에게 전화할 필요가 없다.
소재 검수 실패 사유를 읽은 사람이 제작사에 정확히 전달할 수 있다.

### 제약조건
- 라이트 모드 전용 (다크 모드 추후)
- React/Next.js/TypeScript, 상태관리 Jotai + TanStack Query
- 타이포그래피: Pretendard 또는 Noto Sans KR
- 색상: 초록 계열 Primary (#03C75A), AI Slop 색상(인디고·퍼플·네온) 배제
- 광고주 계정 없음 — 4개 역할만 (어드민, 매체사, 운영대행사, 영업대행사)
- 1차 릴리즈: 네이버 NAM SSP만 연동, 매체 수십 대 규모

### 설계 순서 권고
1. **소재 관리** — 검수 상태 타임라인 + 실패 사유 + 규격 사전 안내 (기회 #1, #6, #7) — 최유진 pain 해소 + 매체사 검수 압박 완화
2. **역할별 대시보드** — 동기화 상태 위젯 + 역할별 핵심 요약 (기회 #3, #5) — 전체 역할 첫 진입 경험
3. **편성 관리** — 동시 편집 사전 경고 + diff 패널 (기회 #4, #11) + 소재 편성 연결 상태 (기회 #2)
4. **알림 관리** — 채널 분리 + 헬스체크 + 모바일 채널 (기회 #8, #9, #15)
5. **감사 이력 + 리포트** — 정산 연계 편성 이력 (기회 #10)
6. **2차 릴리즈** — 긴급 편성 프리셋, 소재 버전 롤백, 매체 아카이브 (기회 #12, #13, #14)
