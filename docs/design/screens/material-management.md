## S-12: 소재 목록

### 화면 목적
소재의 검수·운영·편성 상태를 한눈에 파악하고, 업로드·필터·상세 진입의 허브 역할을 한다.

### 레이아웃

```
┌──────────────────────────────────────────────────────┐
│ GNB (56px)                              [bell] [프로필] │
├────────┬─────────────────────────────────────────────┤
│ LNB    │ ① 소재 현황 요약 위젯 (영업대행사만)          │
│ 240px  │   검수중 3 · 검수완료 12 · 검수실패 1 · 만료임박 2 │
│        │─────────────────────────────────────────────│
│ 소재목록│ ② Page Header                               │
│ 규격안내│   "소재 관리"                    [+ 소재 업로드] │
│        │─────────────────────────────────────────────│
│        │ ③ Filter Bar                                │
│        │   검수상태 | 운영상태 | 매체 | 기간 | [검색]    │
│        │─────────────────────────────────────────────│
│        │ ④ Toolbar                                   │
│        │   전체 N건  정렬: 최신순 ▼   [리스트|썸네일]    │
│        │─────────────────────────────────────────────│
│        │ ⑤ Table (리스트 뷰)                          │
│        │   □ 썸네일 | 소재명 | 광고주 | 매체 | 해상도   │
│        │     | 재생시간 | 검수상태 | 편성연결 | 운영상태  │
│        │     | 등록일                                  │
│        │   ─────────────────────────────────────────  │
│        │   row row row ...                            │
```

썸네일 뷰 전환 시 ⑤영역이 4열 카드 그리드(썸네일 + 소재명 + 검수배지 + 운영배지)로 교체.

### 컴포넌트 목록

| 컴포넌트 | 설명 | 상태 | 토큰 참조 |
|---------|------|------|----------|
| SummaryWidget | 영업대행사 전용 상태별 소재 수 카운터. 카드 4칸 | default | `--color-bg-surface`, `--radius-lg`, `--shadow-sm` |
| PageHeader | 타이틀 + primary action | — | `--text-xl`, `--space-10` 상하 패딩 |
| FilterBar | 공통 컴포넌트 (하단 참조) | — | — |
| ViewToggle | 리스트/썸네일 아이콘 토글 | active/inactive | `--color-interactive-primary` active, `--color-text-tertiary` inactive |
| DataTable | 정렬 가능 컬럼. 행 호버 배경 | default/hover/selected | `--color-bg-hover`, `--text-base` |
| ThumbnailCard | 16:9 썸네일 + 메타 2줄 + 배지 | default/hover | `--radius-lg`, `--shadow-md` hover |
| StatusBadge | 공통 컴포넌트 (하단 참조) | 검수 4종 + 편성 4종 + 운영 3종 | — |
| Pagination | 페이지 번호 + 페이지당 건수 선택 | — | `--text-sm` |

### 역할별 차이

| 역할 | 데이터 범위 | UI 차이 |
|------|-----------|---------|
| 영업대행사 | 본인 등록 소재 (공유 설정된 소재 포함) | ① 소재 현황 요약 위젯 상단 표시. 광고주 필터 추가. 매체 필터 미노출 |
| 매체사 | 본인 매체사 전체 소재 | 매체 필터 표시. 수동검수대기 배지 카운터 필터바 옆 표시 |
| 운영대행사 | 부여받은 매체 소재 | 매체사 컨텍스트 스위처(GNB)로 범위 전환 |
| 어드민 | 전체 소재 | 수동검수대기 카운터 배지 (FilterBar 옆, `--color-status-warning` 배경) |

### 상태별 표시

**검수 상태** (StatusBadge):

| 상태 | 배지 | 행 처리 |
|------|------|---------|
| 검수중 | spinner + "검수중 ~N분" | 기본 |
| 검수완료 | green check | 기본 |
| 검수실패 | red X + 사유 1줄 truncate | 기본 |
| 수동검수대기 | orange clock + "수동검수대기" | 기본 |

**편성 연결 상태** (텍스트 라벨):
미편성 (`--color-text-tertiary`) / 편성대기 (`--color-text-secondary`) / 송출중 (`--color-status-success`) / 송출종료 (`--color-text-tertiary`)

**운영 상태**:
예정 (neutral label) / 진행중 (기본) / 만료 (행 전체 `opacity: 0.5`, `--color-text-tertiary`)

### 인터랙션
- Row click → S-13 소재 상세
- [+ 소재 업로드] → Upload Drawer (공통 컴포넌트)
- SummaryWidget 카운터 클릭 → 해당 상태로 필터 적용
- 필터 값은 URL query param에 반영 (뒤로가기 시 복원)
- 정렬: 등록일, 소재명, 검수상태, 운영상태

### 빈 상태
- **신규 사용자**: 일러스트 + "아직 등록된 소재가 없습니다." + [소재 업로드] 버튼
- **필터 결과 없음**: "조건에 맞는 소재가 없습니다." + [필터 초기화] 텍스트 링크

## S-13: 소재 상세

### 화면 목적
소재의 검수 진행 상황, 실패 사유와 수정 가이드, 편성 연결 현황을 확인하고 교체·삭제를 수행한다.

### 레이아웃

```
┌──────────────────────────────────────────────────────┐
│ GNB                                                   │
├────────┬─────────────────────────────────────────────┤
│ LNB    │ Breadcrumb: 소재 관리 > 소재명                │
│        │─────────────────────────────────────────────│
│        │ ┌─────────────┬───────────────────────────┐ │
│        │ │ ⓐ Preview   │ ⓑ Metadata & Status      │ │
│        │ │             │                           │ │
│        │ │ [thumbnail] │ 소재명: xxx               │ │
│        │ │  ▶ Play     │ 광고주: xxx               │ │
│        │ │             │ 매체: xxx                 │ │
│        │ │ 파일 정보    │ 해상도: 1920×1080         │ │
│        │ │ MP4 H.264   │ 재생시간: 15초            │ │
│        │ │ 30fps       │ 운영기간: 04.01~04.30     │ │
│        │ │ 12.4 MB     │ 등록일: 2026-04-01 14:32  │ │
│        │ │             │                           │ │
│        │ │             │ ⓒ 상태 타임라인            │ │
│        │ │             │ ● 업로드 04.01 14:32      │ │
│        │ │             │ ● 검수중 (~3분)           │ │
│        │ │             │ ◉ 검수완료 04.01 14:35    │ │
│        │ │             │                           │ │
│        │ │             │ Actions: [교체] [삭제]     │ │
│        │ └─────────────┴───────────────────────────┘ │
│        │─────────────────────────────────────────────│
│        │ ⓓ 편성 연결 정보                             │
│        │ ┌─────────────────────────────────────────┐ │
│        │ │ 매체명 | 편성표명 | 편성상태 | 기간        │ │
│        │ │ 강남역 | 4월1주차 | 송출중   | 04.01~07  │ │
│        │ │ 종로   | 4월광고  | 편성대기 | 04.08~14  │ │
│        │ └─────────────────────────────────────────┘ │
│        │─────────────────────────────────────────────│
│        │ ⓔ 버전 이력 (접힘)                    [펼치기]│
└────────┴─────────────────────────────────────────────┘
```

ⓐ Preview: 5col / ⓑ Metadata+Status: 7col (12-column grid 기준)

### 상태 타임라인 (ⓒ)

수직 스텝 인디케이터. 각 노드에 타임스탬프와 소요 시간 표시.

```
● 업로드완료  04.01 14:32
│
● 검수중      04.01 14:32  예상 ~3분
│
◉ 검수완료    04.01 14:35  (소요 3분)
```

| 노드 상태 | 아이콘 | 색상 |
|----------|--------|------|
| 완료 | filled circle | `--color-status-success` |
| 현재 진행 | spinner ring | `--color-interactive-primary` |
| 실패 | X circle | `--color-status-error` |
| 대기 | clock circle | `--color-status-warning` |
| 미도달 | empty circle | `--color-border-default` |

### 검수 실패 UX

타임라인의 실패 노드 아래에 인라인 확장:

```
✕ 검수실패  04.01 14:38
  ┌────────────────────────────────────────┐
  │ 사유: 해상도가 다릅니다                   │
  │       등록: 1920×1080, 실제: 1280×720   │
  │                                        │
  │ 수정 가이드:                             │
  │ "1920×1080 해상도로 다시 인코딩 후        │
  │  업로드해주세요"                          │
  │                                        │
  │ [수정 가이드 복사]          [재업로드]     │
  └────────────────────────────────────────┘
```

- **사유**: 비기술적 표현. "codec mismatch" 대신 "영상 형식이 맞지 않습니다"
- **[수정 가이드 복사]**: 소재명 + 사유 + 가이드만 포함한 클린 텍스트 복사. 다른 소재 정보, 광고주명, 시스템 URL 미포함 (타 광고주 정보 노출 방지)
- **[재업로드]**: Upload Drawer 열기. 기존 메타데이터 프리필

### 소재 교체 UX
1. [교체] 클릭 → Upload Drawer (파일 선택만, 메타데이터 유지)
2. 편성 사용중 소재: 확인 모달 — "이 소재는 N건의 재생목록에 사용 중입니다. 교체하면 검수 완료 후 자동 반영됩니다. 계속하시겠습니까?"
3. 업로드 시작 → toast: "기존 소재가 재검수 완료까지 유지됩니다. 예상 소요: ~N분"
4. 타임라인에 "교체중" 배지 표시. 편성 연결 테이블에서도 해당 구좌 "교체중" 표시
5. 재검수 완료 → 신규 파일로 자동 전환. 재검수 실패 → 기존 파일 유지, 알림

### 삭제 UX
- [삭제] 클릭 → 확인 모달:
  - 편성 미사용: "이 소재를 삭제하시겠습니까?"
  - 편성 사용중: "이 소재를 삭제하시겠습니까? 재생목록 N건에서 해당 구좌가 '삭제됨'으로 표시됩니다." (`--color-status-error` 강조)
- 모달 버튼: [취소] (secondary) / [삭제] (destructive, `--color-status-error` 배경)
- Soft delete — 목록에서 숨김. "삭제됨" 필터로 접근 가능

### 편성 연결 정보 (ⓓ)

| 컬럼 | 설명 |
|------|------|
| 매체명 | 송출 매체 |
| 편성표명 | 연결된 편성표 |
| 편성 상태 | 미편성 / 편성대기 / 송출중 / 송출종료 (StatusBadge) |
| 기간 | 편성 적용 기간 |

영업대행사: 읽기 전용 (편성표 링크 없음, INV-13). "내 소재가 어디서 나가는지" 확인용.
매체사/어드민: 편성표명 클릭 → S-21 편성표 상세 이동.

### 버전 이력 (ⓔ)

접힌 상태 기본. 교체 이력 존재 시에만 표시. 최대 5버전 보존.

| 버전 | 파일명 | 교체일 | 검수결과 | 액션 |
|------|--------|--------|---------|------|
| v2 (현재) | ad_v2.mp4 | 04.03 | 검수완료 | — |
| v1 | ad_v1.mp4 | 04.01 | 검수완료 | [되돌리기] |

[되돌리기] → 확인 모달 → 해당 버전으로 교체 + 재검수

### 역할별 차이

| 역할 | 편집 가능 필드 | 액션 |
|------|--------------|------|
| 영업대행사 | 비고만 | 업로드, 교체, 삭제. 편성 연결은 읽기 전용 |
| 매체사 | 전체 메타데이터 | 전체 + 수동검수 처리 (UC-15) |
| 어드민 | 전체 메타데이터 | 전체 + 수동검수 처리 |
| 운영대행사 | 권한 범위 내 메타데이터 | 업로드, 교체, 삭제 |

## S-14: 소재 규격 안내

### 화면 목적
광고주/제작사에 사전 공유할 수 있는 매체별 소재 규격 가이드. 검수 실패율을 사전에 낮춘다.

### 레이아웃

```
┌──────────────────────────────────────────────────────┐
│ GNB                                                   │
├────────┬─────────────────────────────────────────────┤
│ LNB    │ Page Header: "소재 규격 안내"    [링크 복사]   │
│        │─────────────────────────────────────────────│
│        │ 공통 규격                                     │
│        │ ┌─────────────────────────────────────────┐ │
│        │ │ 파일 형식: MP4 (H.264 코덱)              │ │
│        │ │ 프레임레이트: 30fps                       │ │
│        │ │ 최대 파일 크기: (미정)                     │ │
│        │ └─────────────────────────────────────────┘ │
│        │─────────────────────────────────────────────│
│        │ 매체별 규격                                   │
│        │ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│        │ │ 강남역    │ │ 종로     │ │ 홍대입구  │     │
│        │ │ 1920×1080│ │ 3840×1080│ │ 1920×1080│     │
│        │ │ 16:9     │ │ 32:9     │ │ 16:9     │     │
│        │ │ 15초/30초│ │ 15초     │ │ 15초/30초│     │
│        │ └──────────┘ └──────────┘ └──────────┘     │
└────────┴─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 설명 | 토큰 참조 |
|---------|------|----------|
| CommonSpecCard | 공통 규격 카드 (포맷, 코덱, fps) | `--color-bg-surface`, `--radius-lg`, `--space-6` 패딩 |
| MediaSpecCard | 매체별 규격 카드. 매체명, 해상도, 비율, 허용 재생시간 | `--color-bg-elevated`, `--radius-lg`, `--shadow-sm` |
| CopyLinkButton | 공유 URL 클립보드 복사. 토큰 기반 또는 public URL | `--color-interactive-primary` |

### 공유 방식
- [링크 복사] → 클립보드에 URL 복사 + toast "링크가 복사되었습니다"
- 공유 URL은 로그인 불필요 (토큰 기반 공개 링크 또는 public 페이지)
- 영업대행사가 광고주/제작사에 카톡으로 전달하는 워크플로우 대응

## 공통 컴포넌트

### Upload Drawer

화면 우측에서 슬라이드 인. `--transition-slow` (300ms).

```
┌─── Upload Drawer (480px) ──────────────┐
│ 소재 업로드                      [닫기] │
│────────────────────────────────────────│
│ ┌────────────────────────────────────┐ │
│ │                                    │ │
│ │   파일을 드래그하거나 [찾아보기]      │ │
│ │   MP4 · H.264 · 30fps             │ │
│ │                                    │ │
│ └────────────────────────────────────┘ │
│────────────────────────────────────────│
│ 파일 목록 (복수 가능)                   │
│  ☑ ad_01.mp4  12.4MB  ✓ 검증통과      │
│  ☑ ad_02.mp4  8.1MB   ✓ 검증통과      │
│  ☐ ad_03.avi  —       ✕ MP4만 허용    │
│────────────────────────────────────────│
│ 메타데이터 (파일별 / 일괄 적용 토글)     │
│  소재명*     [____________]            │
│  광고주      [____________]            │
│  대상 매체*  [▼ 매체 선택]              │
│  해상도*     [▼ 해상도 선택]            │
│  운영 기간*  [시작일] ~ [종료일]         │
│  비고        [____________]            │
│────────────────────────────────────────│
│ [일괄 적용] 체크 시: 모든 파일에 동일     │
│  메타데이터 적용                         │
│────────────────────────────────────────│
│              [취소]  [업로드 (N건)]      │
└────────────────────────────────────────┘
```

**인라인 검증** (파일 선택 즉시, 업로드 전):
- 파일 형식 ≠ MP4 → ✕ "MP4 파일만 업로드 가능합니다"
- 코덱 ≠ H.264 → ✕ "H.264 코덱의 MP4 파일만 허용됩니다. 현재: [코덱명]"
- fps ≠ 30 → ✕ "30fps MP4 파일만 허용됩니다. 현재: [N]fps"
- 검증 실패 파일은 업로드 대상에서 자동 제외 (체크 해제, 비활성)

**업로드 진행**: 파일별 progress bar. 완료 시 자동으로 "검수중" 상태 전환.
