## 0. 매체 상태 열거형 (Canonical Status Enum)

S-02-dashboard-admin.md 섹션 0과 동일한 단일 출처 enum을 따른다. 두 파일 간 불일치 발생 시 admin spec이 정본이다.

| 값 | 표시명 | 색상 토큰 | 의미 |
|----|--------|----------|------|
| `online` | 온라인 | `--color-interactive-hover` (primary-600) | 정상 동작 중 |
| `delayed` | 지연 | `--color-status-warning` (warning-500) | 동기화 지연 또는 응답 지연 |
| `error` | 이상 | `--color-status-error` (error-500) | 오프라인 또는 이상 감지 |
| `offline` | 오프라인 | neutral-400 | 연결 끊김 (에러 아님, 의도적 오프라인 포함) |
| `inactive` | 비활성 | neutral-300 | 운영 중지 매체 |

## 1. 개요

운영대행사 전용 대시보드. 이준혁(주니어 매니저)이 담당하는 **3개 매체사·150대 매체**를 동시에 파악하고 "지금 어떤 매체사에서 뭘 처리해야 하나"라는 질문에 즉시 답하는 것이 주목적.

핵심 페르소나 특성: 기술 이해도 낮음, 실수 공포 큼, 판단을 매체사에 위임하는 경향. 따라서 **처리 필요 항목을 우선순위별로 전면 배치**하고 드릴다운 경로를 명확히 한다.

어드민·매체사 대시보드와 레이아웃 패턴을 공유하되 다음을 차별화한다:
- **추가**: GNB 컨텍스트 스위처 탭, 작업 큐 히어로, 매체사별 현황 섹션
- **제거 (전체 탭 뷰 기준)**: 동기화 히어로 도넛, 소재 검수 상세 카드, 시스템 상태 섹션, Slack 항목 — 개별 탭 뷰에서는 매체사 대시보드 레이아웃을 그대로 사용하므로 이 항목들이 다시 등장함 (섹션 4 참조)
- **변경**: 지도 마커가 담당 매체사 전체(전체 탭) 또는 선택 매체사(개별 탭)로 필터링

## 3. 컴포넌트

### 사이드바

- 배경: `oklch(0.16 0.02 155)` (다크 그린-틴티드)
- 너비 240px, 좌측 고정
- 로고 영역: 28×28 브랜드 마크 + "Bari CMS" (`--text-sm`, 700, white)

**GNB 항목 (운영대행사 역할 — 6개):**
대시보드 / 매체 관리 / 소재 관리 / 캠페인 관리 / 편성 관리 / 리포트

- active: 배경 `oklch(1 0 0 / 0.08)`, 우측 2px border `--color-interactive-primary`
- 비활성: `oklch(1 0 0 / 0.55)` 텍스트
- 알림 뱃지 없음 (운영대행사는 알림 센터 GNB 항목 없음 — IA 기준 6개)

**LNB 서브항목 (GNB 항목 선택 시 확장):**
- 대시보드: 서브항목 없음
- 매체 관리: 매체 목록(S-06) — S-11 매체 그룹은 운영대행사 접근 불가 (IA 역할 매트릭스 ❌)
- 소재 관리: 소재 목록(S-12)
- 캠페인 관리: 캠페인 목록(S-15)
- 편성 관리: 재생목록(S-18) / 편성표(S-20) / 싱크 송출(S-23) / 잔여 구좌 조회(S-25)
- 리포트: 리포트 목록(S-27) / 리포트 생성(S-28)

현재 화면이 대시보드이므로 LNB 서브항목 미표시.

**하단 고정:** 역할명 "운영대행사" + 사용자명 "이준혁". 클릭 → S-29 개인 설정

### GNB (상단 바) — 컨텍스트 스위처 포함

- 배경: white, 하단 border `--color-border-default`
- 좌: 페이지 제목 "대시보드" (`--text-md`, 700)
- 중앙: **컨텍스트 스위처 탭 그룹**
  - 탭 순서: 전체 (기본) / [담당 매체사명 동적 나열 — 최대 5개]
  - 탭 스타일: pill-tab (pill 형태, 배경 `oklch(0.94 0.005 80)`)
  - active 탭: 배경 white, 그림자 `shadow-sm`, 텍스트 `--color-text-main`
  - 비활성 탭: 텍스트 `--color-text-sub`
  - "전체" 탭 선택 시: 담당 매체사 전체 집계 뷰
  - 개별 탭 선택 시: 해당 매체사 단독 뷰 (섹션 4 참조)
  - **엣지 케이스 — 담당 매체사 1개**: "전체" 탭 미표시, 해당 매체사 탭만 단독으로 표시 (집계 뷰 의미 없음). 바로 단독 뷰로 진입
  - **엣지 케이스 — 매체사명 길이**: 탭 레이블 최대 8자, 초과 시 "…" 말줄임. 풀네임은 hover 시 툴팁으로 표시
  - **엣지 케이스 — 담당 매체사 6개 이상**: 탭 영역 overflow 시 우측 스크롤 화살표 표시
- 우: 날짜 표시 + 알림 벨 아이콘 (미읽음 도트)

### 작업 큐 히어로 (전체 탭 기본 뷰)

운영대행사 대시보드의 최우선 시각 요소. "지금 처리 필요" 섹션으로 이준혁의 핵심 질문에 즉시 답한다.

**레이아웃:** 가로 4칸 균등 그리드

| 칸 | 내용 | 우선순위 색상 |
|----|------|--------------|
| 이상 매체 | `error` + `offline` 상태 합산 수 | 빨강 (`--color-status-error`) |
| 검수 대기 | 소재 검수 대기 건수 (전체 매체사 합산) | 노랑 (`--color-status-warning`) |
| 편성 예정 | 오늘 적용 예정 편성표 건수 | 노랑 (`--color-status-warning`) |
| 정상 매체 | `online` 상태 매체 수 | 초록 (`--color-interactive-primary`) |

**각 칸 구성:**
- 숫자: `--text-2xl`, weight 800, 해당 우선순위 색상
- 레이블: `--text-xs`, `--color-text-sub`
- 서브텍스트: 매체사별 분포 (예: "B사 2 · A사 2"), `--text-xs`, `--color-text-sub` — **이상 없는 칸은 서브텍스트 생략**
- 배경: 각 우선순위 색상 5% 투명도 (`oklch(.../ 0.05)`), 1px border 해당 색상 15% 투명도
- 클릭: 이상 매체 → 매체 목록(S-06) 이상 필터, 검수 대기 → 소재 목록(S-12), 편성 예정 → 편성 관리, 정상 매체 → 매체 목록(S-06)

**섹션 헤더:** "지금 처리 필요 — [N]개 매체사 · [총대수]대" + 우측 "N분 전 갱신" (`--text-xs`, `--color-text-sub`)

### 매체사별 현황 섹션

작업 큐 히어로 아래 배치. 담당 매체사를 한 행씩 나열해 어느 회사에 이상이 있는지 즉시 파악할 수 있도록 한다.

**정렬 기준:** `error` + `offline` 수 내림차순 → 동률 시 `delayed` 수 내림차순

**각 행 구성 (좌→우):**
1. **컬러 인디케이터** (3×30px 세로 바): 이상 있으면 `--color-status-error`, 이상 없으면 `--color-interactive-primary`
2. **매체사 정보**: 매체사명 (`--text-sm`, 600) + 총 매체 수 (`--text-xs`, `--color-text-sub`). 이상 있으면 매체사명 옆 `⚠` 아이콘 표시
3. **오늘 작업 요약**: 소재 검수 대기 N건 + 편성 충돌 있으면 "편성 충돌 N건" 경고 표시 (있을 때만). 모두 정상이면 미표시. (`--text-xs`, `--color-text-sub`)
4. **상태 칩**: `online N` / `error N` / `delayed N` — 해당 상태가 0이면 칩 미표시. 색상은 Canonical Status Enum 기준
5. **화살표** `›`: 클릭 트리거 표시

행 클릭 → 해당 매체사 컨텍스트 탭으로 전환 (단독 뷰)

### 하단 2열 그리드

**최근 알림 패널 (좌):**
- 섹션 제목 "최근 알림" + 우측 "알림 센터 →" 링크 (단, 운영대행사는 알림이 자사 담당 매체사 범위로 필터링)
- 알림 항목: 미읽음 도트 (primary) / 읽음 도트 (neutral-300), 알림 텍스트, 시간
- 알림 텍스트에 **매체사명 접두사** 필수 (예: "B사 강남구 3번 오프라인 감지")

**캠페인/편성 패널 (우):**
- 섹션 제목 "캠페인 / 편성" + 우측 필터 드롭다운 (기간: 최근 7일/30일)
- 집행중 캠페인 수 + 진행률 표시줄
- 오늘 예정 편성 수 + 진행률 표시줄
- 편성 충돌 상태 표시 (충돌 없음 / 충돌 N건)

## 4. 개별 매체사 탭 뷰 (단독 뷰)

GNB에서 "A사 미디어" 등 개별 탭 선택 시, 좌측 스크롤 패널 전체를 **해당 매체사 단독 뷰**로 교체한다. 사이드바는 운영대행사 GNB 6개 항목 유지.

**렌더링 섹션 (매체사 대시보드 S-02-dashboard-media-company.md 동일 구조 사용):**

| 섹션 | 내용 | 비고 |
|------|------|------|
| 이상 감지 배너 | 해당 매체사 이상/지연 알림 | 이상 없으면 미표시 |
| 동기화 히어로 도넛 | 해당 매체사 매체만 집계 | S-02-dashboard-media-company.md 섹션 3 동일 |
| 미동기화 매체 섹션 | 해당 매체사 error/delayed/offline 매체만 표시 | |
| 소재 검수 현황 | 해당 매체사 소재만 | |
| 캠페인/편성 + 알림 2열 | 해당 매체사 범위 | |

**지도:** 해당 매체사 마커만 표시. 지도 헤더 "[매체사명]" 표시.

**탭 전환 동작:**
- 탭 클릭 시 좌측 패널만 교체 (사이드바·GNB 유지)
- 전환 중 스켈레톤 로딩 (카드 높이 유지, 내용 shimmer)
- 전환 실패 시 패널 내 인라인 오류 메시지 표시 (배너가 아닌 패널 중앙)

**"전체" 탭 복귀:** 전체 탭 클릭 시 집계 뷰로 즉시 전환 (캐시된 데이터 우선 표시 후 백그라운드 갱신)

## 5. 지도 패널

**헤더:** "매체 위치 — [탭명]" + 우측 "[총 매체 수]대"
- 전체 탭: "매체 위치 — 전체" + "150대"
- 개별 탭: "매체 위치 — A사 미디어" + "50대"

**마커 필터링:**
- 전체 탭: 담당 3개 매체사 전체 마커 표시
- 개별 탭: 해당 매체사 마커만 표시

**마커 툴팁:** "[매체사명] [위치명] · [상태]"

**범례:** 온라인 / 지연 / 이상 (3가지 — offline/inactive는 마커 미표시 또는 neutral 색상)

**지도 구현:** 실제 서비스 시 Naver Maps SDK. 프로토타입에서는 CSS gradient로 시뮬레이션.
