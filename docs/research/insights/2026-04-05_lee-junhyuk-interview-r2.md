## 인터뷰 트랜스크립트

### 1. 현장 디바이스 연동 경험 (02 UC-08)

**Q: 디바이스 교체 작업을 현장에서 직접 하시나요? 어떤 절차로 진행되는지 설명해주세요.**

> 네, 장비 고장나면 보통 현장 기사가 새 플레이어를 물리적으로 설치하고, 저는 CMS에서 연동 작업을 해요. 근데 문제는 현장 기사가 새 장비의 디바이스 ID를 카톡으로 사진 찍어 보내주거든요. 그 사진이 각도가 안 좋거나 화질이 구려서 O랑 0, I랑 1, 8이랑 B를 헷갈릴 때가 많아요. 한 번은 ID를 세 번 잘못 입력해서 "유효하지 않은 디바이스 ID입니다"만 계속 뜨고, 결국 현장 기사한테 다시 전화해서 한 글자씩 불러달라고 했어요.

**Q: 디바이스 ID 오입력으로 다른 매체에 연동된 적은요?**

> 아... 한 번 있었어요. 숫자 하나 잘못 입력했는데 "이 디바이스는 다른 매체에 연동되어 있습니다"라고 떠서, 그때는 아 내가 잘못 입력했구나 바로 알았죠. 근데 진짜 무서운 건, 유효한 ID인데 다른 장비의 ID를 입력하면 어떻게 되는 거예요? 그러니까 A 매체에 B 장비를 연동해버리는 거죠. 그러면 A 매체의 편성이 엉뚱한 장소에서 나가는 건데... 이걸 시스템이 잡아주나요? 저는 잘 모르겠어요.

**Q: 영상 출력 조정(사이즈/위치)은 어떤 식으로 하세요?**

> 그게 진짜 스트레스예요. 현장에서 LED 패널 앞에 서서 방향키로 조정하는데, 기준이 없어요. 직접 눈으로 보면서 "이 정도면 됐나?" 하는 거죠. 그런데 LED가 건물 5층 높이에 붙어있으면 밑에서 올려다보는 거라 왜곡이 있어요. 조정하고 나서 팀장님한테 사진 찍어 보내면 "좀 더 왼쪽" 이러시고... 두세 번 왔다갔다 해요. 한 번은 잠금 안 누르고 돌아왔는데, 다음 날 현장 기사가 뭔가 건드려서 위치가 틀어져 있었어요. 광고주가 "우리 광고 잘려서 나간다"고 전화 왔었죠.

**Q: 조정할 때 CMS 화면에서 미리보기 같은 건 없나요?**

> 없어요. 그냥 현장에서 실물 보면서 하는 거예요. 사무실에서는 어떻게 나가는지 확인할 방법이 없어요. 그래서 항상 현장에 가야 하고, 왕복 시간 포함하면 매체 하나에 2~3시간 걸려요. 150대 관리하는데 디바이스 교체가 한 달에 5~6건은 되니까... 상당한 시간이에요.

### 2. 편성 관리 일상 (05 UC-21~25)

**Q: 3개 매체사 편성을 동시에 관리하는 하루 일과를 말씀해주세요.**

> 아침에 출근하면 브라우저 탭 세 개 띄워요. 매체사 A, B, C 각각. 그런데 로그인을 세 번 해야 해요. 계정이 매체사별로 다르니까. 아이디 비번도 다 다르고, 가끔 비번 바뀌면 또 까먹고... 오전에 보통 전날 저녁에 온 편성 요청을 처리하고, 오후에는 다음 주 편성을 미리 세팅해요. 근데 매체사 A 작업하다 중간에 매체사 B에서 급한 건 들어오면 탭 전환하잖아요? 그때 머릿속 컨텍스트가 섞여요. "아까 어디까지 했더라?" 이런 거.

**Q: 편성 충돌이 나면 어떻게 해결하세요?**

> 솔직히 우선순위 규칙을 정확히 모르겠어요. 1순위가 뭐고 3순위가 뭔지... 저장하려고 하면 "충돌" 뜨는데, 거기서 옵션이 나오잖아요. 근데 "우선순위를 올려서 기존 편성표를 대체"라는 게, 기존 거를 없애는 건지 밑으로 가는 건지 헷갈려요. 그래서 항상 매체사 담당자한테 카톡으로 스크린샷 보내고 "이거 어떻게 할까요?" 물어봐요. 담당자가 바빠서 답이 늦으면 그냥 기다려요. 제가 잘못 건드리면 큰일나니까.

**Q: "즉시 반영" 버튼에 대해서는 이전에도 얘기하셨는데, 지금은 어떠세요?**

> (한숨) 지금은 알죠. 근데 아직도 가끔 까먹어요. 특히 급할 때. 편성표 저장하고 "됐다" 하고 다음 작업 넘어가는데, 한 시간 뒤에 매체사에서 전화 와요. "편성 안 바뀌었는데요?" 그때 아차 싶은 거죠. 가장 답답한 건, 적용중인 편성표를 수정했을 때예요. 저장만 하면 현장에서는 옛날 거가 계속 나가고 있거든요. 근데 편성표 목록에서는 수정된 내용으로 보여요. CMS 화면이랑 현장이 다른 거죠. 이게 진짜 혼란스러워요.

**Q: 재생목록이랑 편성표가 별개인 건 이해하고 계시죠?**

> 음... 솔직히 처음에 되게 헷갈렸어요. "재생목록을 먼저 만들고 편성표에 연결한다"는 게 왜 두 단계인지 이해가 안 됐거든요. 그냥 편성표에서 바로 소재 넣으면 안 되나? 하는 생각이었어요. 지금은 익숙해졌지만, 신입이 들어오면 저도 설명하기가 어려워요. "재생목록은 재생할 소재 순서고, 편성표는 그걸 어떤 매체에 언제 적용하는 거야"라고 하면 다들 "그냥 편성표에서 다 하면 안 돼요?" 이래요.

### 3. 동시 수정 문제 (낙관적 잠금)

**Q: 다른 사용자와 동시에 같은 편성표를 수정한 경험을 자세히 말씀해주세요.**

> 가장 큰 사고가 지난달이에요. 매체사 A의 주말 편성을 제가 작업하고 있었는데, 구좌가 20개짜리 재생목록이었어요. 소재를 7개 교체하고 순서도 다 바꿨어요. 30분은 걸렸을 거예요. 저장 눌렀더니 "다른 사용자가 이 편성표를 수정했습니다" 뜨는 거예요. 매체사 A 김 대리가 같은 편성표에서 구좌 하나만 바꿨더라고요. 근데 그게 먼저 저장돼서 제 30분 작업이 날아간 거죠.

**Q: "비교 패널"에서 최신 내용을 확인하셨어요?**

> 아 뭔 비교 패널이요? 그냥 에러 메시지만 떴어요. 지금 CMS에 그런 기능 없어요. 만약 새 CMS에 비교 패널이 생긴다면... 글쎄요, 도움이 되긴 하겠지만, 제 입장에서는 "비교"보다 "안 날아가는 게" 중요해요. 저장 누르기 전에 다른 사람이 수정 중이라고 알려주면 훨씬 낫죠. "지금 김 대리님이 이 편성표를 보고 있습니다" 이런 거요. 그러면 전화해서 "저 먼저 할게요" 하거나 기다리거나 할 수 있잖아요.

**Q: 한 달에 이런 일이 몇 번 있나요?**

> 편성 덮어쓰기까지 간 건 월 1~2회 정도? 근데 "아, 다른 사람이 수정했네" 하고 새로고침해서 다시 하는 건 주 2~3회는 되는 것 같아요. 매체사 담당자랑 동시에 작업하는 거니까 어쩔 수 없긴 한데... 좀 화나죠.

### 4. 매체 상태 모니터링 (02 UC-11)

**Q: 플레이어 동기화 상태를 어떻게 확인하세요?**

> CMS에서 매체 모니터링 탭 들어가면 "온라인/오프라인" 나오는 건 알아요. 근데 그게 "온라인이면 편성이 잘 나가고 있다"는 뜻인 줄 알았는데, 아니더라고요. 온라인인데 편성이 안 바뀌어 있는 경우가 있어요. "동기화 미완료" 같은 건 못 봤는데... 아마 있을 수도 있는데 제가 모르는 걸 수도?

**Q: 현장 상태를 직접 확인하러 가시는 경우는?**

> 광고주가 "우리 광고 나가고 있는 거 맞아요?" 하면 확인해야 해요. 근데 150대 중에 서울 안에 있는 건 직접 가보기도 하지만, 지방은 못 가잖아요. 그때는 현장 기사한테 부탁해서 사진 찍어달라고 해요. 진짜 원시적이죠. CMS에서 실시간으로 "지금 무슨 광고 나가고 있다" 볼 수 있으면 좋겠어요. 김서영 팀장님도 원격 스크린샷 얘기하시더라고요.

**Q: 매체가 오프라인 됐을 때 알림은 잘 받으시나요?**

> 알림이 오긴 하는데... 150대 관리하다 보면 알림이 너무 많아요. 특히 새벽에 업데이트하면서 잠깐 오프라인 됐다 복구되는 것도 다 알림이 오거든요. 진짜 중요한 — 예를 들어 낮 시간에 오프라인 된 건 빨리 봐야 하는데, 새벽 업데이트 알림에 묻혀서 못 볼 때가 있어요. 아침에 출근해서 알림 30개 쌓여있으면 그냥 다 읽음 처리하고 넘어가요. 그러다 진짜 장애를 놓치는 거죠.

### 5. 소재 관리 (03 UC-13, 16)

**Q: 급한 소재 교체 요청이 들어오면 어떻게 하세요?**

> 광고주가 "오늘 오후부터 새 소재로 바꿔달라"고 하면, 일단 소재를 올려요. 근데 검수가 있잖아요. 자동 검수가 보통 얼마나 걸리는지 모르겠는데, 급할 때는 1분도 길어요. "검수중" 상태에서 기다리면서 편성 작업을 미리 하고 싶은데, 검수완료 안 된 소재는 재생목록에 못 넣잖아요. 그래서 검수 끝날 때까지 그냥 화면 새로고침하면서 기다려요. F5 누르면서. 끝나면 부랴부랴 재생목록 수정하고, 편성표 수정하고, "즉시 반영" 누르고... 이 과정이 다 합쳐서 30분은 걸려요. 광고주한테 "오후 2시부터"라고 약속했는데 2시 반에 되는 거죠.

**Q: 소재 교체 중에 기존 파일이 계속 재생되는 건 아시나요?**

> 네? 교체 중에도 옛날 거가 나가요? 아... 그렇구나. 저는 교체 버튼 누르면 바로 새 거로 바뀌는 줄 알았어요. 재검수가 있다는 거죠? 그러면 광고주한테 "지금 바꾸고 있으니까 조금만 기다려주세요" 할 때, 실제로는 기존 광고가 계속 나가는 거잖아요. 그건 좋은 건가... 아니 근데 광고주 입장에서는 "이미 교체 요청했는데 왜 아직 옛날 거 나가냐"고 할 수도 있는 거잖아요. 어느 쪽이든 스트레스예요.

**Q: 소재 업로드할 때 코덱이나 프레임레이트 오류 경험은요?**

> 자주 있죠. 광고주가 보내주는 파일이 가끔 규격 안 맞아요. "H.264 코덱이 아닙니다"라고 뜨면 저는 광고주한테 그대로 전달해요. 근데 광고주가 "그게 뭔 소리예요?" 하면 저도 설명을 못 해요. 제가 영상 전문가가 아니니까. "MP4로 다시 보내주세요"라고 하면 광고주가 "이것도 MP4인데요?" 이러고... 결국 영상 제작사한테 연락하라고 안내하는데, 그 과정에서 시간이 또 가요.

## 페르소나 정의와 다른 응답

- **소재 교체 프로세스 이해 부재**: 페르소나 정의에는 "소재 업로드 매일 수행"이라고 되어있으나, 소재 교체 시 재검수가 발생하고 기존 파일이 유지된다는 점을 전혀 인지하지 못함. "교체 = 즉시 반영"으로 생각하고 있었음. 기술적 이해도 정의를 **"CMS의 비동기 처리 모델(검수, 반영, 동기화)을 이해하지 못하는 수준"**으로 확장 필요
- **알림 과부하**: 페르소나의 고충에 "편성 적용 후 실제 플레이어 재생 상태 확인 불가"가 있으나, 실제로는 확인 자체보다 **알림이 너무 많아 중요한 것을 놓치는 것**이 더 큰 문제. 고충 항목에 "알림 과부하로 인한 장애 인지 지연" 추가 필요
- **편성 충돌 해소 위임**: 정의에 "편성 충돌 시 매체사에 확인 요청"이라고 했으나, 이유가 단순 위험 회피가 아니라 **우선순위 규칙 자체를 이해하지 못해서**임. "1순위가 뭐고 3순위가 뭔지 모른다"는 응답 확보

| 검수 대기 시간이 급한 교체에서 병목 | 확인. 검수완료 전까지 재생목록에 소재 배치 불가 → F5 새로고침 대기 루프 | 03 UC-13~14 |
| 헬스체크 알림이 유용한지 | **부분 확인**. 알림 자체는 유용하나 과부하로 무시 패턴 형성. 새벽 업데이트 알림이 실제 장애 알림을 매몰시킴 | 02 UC-11 |

## 신규 발견

### I-17. 디바이스 ID 입력 UX — QR/바코드 스캔 필요 [상]
- **현상**: 현장 기사가 카톡으로 보낸 사진에서 디바이스 ID를 육안으로 읽고 수동 입력. O/0, I/1, 8/B 혼동으로 평균 2~3회 오입력 후 성공. 유효하지만 잘못된 ID(다른 장비의 ID)를 입력할 경우 엉뚱한 매체에 연동되는 크로스 매칭 위험 존재
- **문제**: UC-08에 "잘못된 디바이스 ID"와 "이미 다른 매체에 연동된 ID" 검증만 있고, **유효하지만 의도하지 않은 장비와의 매칭**을 방지하는 검증이 없음. 현장에서 수동 입력은 에러율이 높음
- **관련**: 02 UC-08 시나리오, MED-BR-01
- **제안**: (1) 플레이어 화면에 QR 코드 표시 → 현장 기사가 스마트폰으로 스캔 → CMS에 자동 입력하는 플로우 추가, (2) 연동 성공 후 "이 장비가 맞습니까? 설치 위치: [주소]" 확인 단계 추가, (3) 연동 직후 테스트 이미지 송출 → 현장에서 육안 확인하는 "연동 확인" 절차 시나리오화

### I-18. 영상 출력 조정 시 원격 미리보기/가이드 레이어 필요 [상]
- **현상**: LED 패널이 건물 5층 높이에 있어 아래에서 올려다보며 조정. 왜곡으로 정확한 정렬 불가. 조정 후 팀장에게 사진 전송 → 피드백 → 재조정 반복. 매체 1대당 2~3시간 소요. 잠금 미설정으로 현장 기사 오조작 사례 확인
- **문제**: UC-08 step 6에 "플레이어에서 디바이스 영상 사이즈와 위치를 방향키로 육안 조정"만 기술. 원격 미리보기나 정렬 기준선이 없어 현장 작업 효율이 극히 낮음. 기존 미결사항에 "가이드 레이어 제공 여부"가 있으나 구체화 안 됨
- **관련**: 02 UC-08, 02 미결 "영상 출력 조정 시 정렬 기준 가이드라인"
- **제안**: (1) CMS에서 실시간 플레이어 화면 미리보기(원격 스크린샷 확장) — 사무실에서 팀장이 직접 확인 가능, (2) 방향키 조정 시 화면에 가이드라인(중심선, 바운더리 박스) 오버레이 표시, (3) 조정 완료 후 자동 잠금 옵션 (잠금 누락 방지)

### I-19. 알림 과부하로 실제 장애 인지 지연 [상]
- **현상**: 150대 매체 관리 시 새벽 플레이어 업데이트 과정에서 일시 오프라인→복구 알림이 대량 발생. 아침 출근 시 알림 30개 이상 적체. 전부 "읽음 처리"하면서 실제 장애(낮 시간 오프라인) 알림도 함께 무시. 광고주 전화로 장애를 인지하는 패턴 반복
- **문제**: UC-11에 알림 발송 정책은 있으나, **알림 우선순위/분류 체계**가 없음. 오프라인→복구를 한 쌍으로 상쇄하는 로직도 없어 불필요한 알림 증폭. 02 미결에 "대량 매체 동시 오프라인 시 알림 집계/그룹핑 정책"이 있으나 미해결
- **관련**: 02 UC-11, 06 알림 관리 도메인
- **제안**: (1) 알림 등급 분류 — 치명(운영 시간 중 오프라인 30분 이상), 경고(운영 시간 중 오프라인 5분 이상), 정보(새벽 업데이트 오프라인→복구). 치명만 푸시, 나머지는 인앱 배지, (2) 오프라인→복구 쌍을 자동 그룹핑하여 "새벽 업데이트 정상 완료 N대" 요약 1건으로 표시, (3) 알림 대시보드에 "현재 오프라인 매체" 실시간 카운터 위젯 — 숫자만 보면 바로 파악

### I-20. 소재 교체 시 재검수 대기 중 "예약 배치" 불가 [중]
- **현상**: 급한 소재 교체 요청 시 — 소재 업로드 → 검수 대기 → 검수완료 확인 → 재생목록 수정 → 편성표 수정 → 즉시 반영. 각 단계가 순차적이라 30분 이상 소요. 검수 대기 중에 재생목록에 "예약"으로 넣어두고 검수 완료되면 자동 반영되는 흐름이 없음
- **문제**: UC-13~16 + UC-21의 흐름에서 INV-11("검수완료 상태의 소재만 편성에 연결 가능")이 엄격하게 적용되어, 검수 중인 소재를 재생목록에 임시 배치할 수 없음. 이로 인해 사용자가 F5 새로고침 루프에 빠짐
- **관련**: 03 UC-13, UC-14, 05 UC-21, INV-11
- **제안**: (1) 재생목록에 "검수 완료 시 자동 배치" 예약 슬롯 기능 추가 — 검수중 소재를 미리 지정하되 실제 편성 반영은 검수완료 후 자동 처리, (2) 검수 상태 변경 시 실시간 알림 + 소재 목록 자동 갱신(F5 불필요), (3) 소재 상세에 예상 검수 완료 시간 표시

### I-21. 편성 우선순위 규칙의 비직관적 라벨링 [중]
- **현상**: "1순위가 제일 높은 건가요 낮은 건가요?"라는 질문에 대답 못함. 1순위=촬영용/특수, 2순위=기간 덮어쓰기, 3순위=기본이라는 체계를 전혀 모름. 매체사 담당자도 "그냥 기본으로 두세요"라고만 안내하여, 이준혁은 항상 3순위(기본)만 사용. 2순위/1순위가 필요한 상황에서도 3순위로 저장 → 충돌 발생 → 매체사에 위임
- **문제**: UC-22 step 7에 우선순위 설정이 있고 UC-24에 충돌 해소가 있으나, **우선순위의 의미를 사용자가 이해하도록 돕는 UI 안내**가 시나리오에 없음. "1순위"라는 숫자 라벨이 실제 용도("촬영용/특수 목적")와 연결되지 않음
- **관련**: 05 UC-22, UC-24, SCH-BR-06
- **제안**: (1) 우선순위 선택 시 용도 설명 인라인 표시 — "1순위 (특수 목적: 촬영, 테스트 등)", "2순위 (기간 한정: 프로모션 등)", "3순위 (기본 편성)" (2) 편성표 생성 시 "이 편성의 목적은?"이라는 질문 형태로 우선순위를 자동 매핑, (3) 충돌 해소 UI에서 우선순위 체계를 시각적 피라미드/다이어그램으로 표시

### I-22. 소재 교체 진행 상태의 비가시성 [중]
- **현상**: 소재 교체 버튼을 눌러도 기존 파일이 계속 재생된다는 사실을 모름("교체 = 즉시 반영"으로 착각). 재검수 진행 상태, 기존 파일 유지 여부, 신규 파일 전환 시점을 CMS에서 직관적으로 파악할 수 없음. 광고주에게 "바꿨습니다"라고 안내한 뒤 실제로는 옛날 소재가 나가는 상황 발생
- **문제**: UC-16 "소재 교체 중 편성 참조 레이스 컨디션"에 "교체중" 상태 표시가 정의되어 있으나, **사용자가 이 상태를 어디서 어떻게 확인하는지** 구체적이지 않음. 소재 상세? 재생목록? 편성표? 어디서 "교체중"이 보이는가?
- **관련**: 03 UC-16, 05 구좌 상태 "교체중"
- **제안**: (1) 소재 교체 시작 시 "기존 소재가 검수 완료까지 유지됩니다. 예상 소요 시간: N분" 안내 토스트, (2) 소재 상세, 재생목록 구좌, 편성표 상세 모두에서 "교체중" 상태를 일관되게 표시, (3) 재검수 완료 → 신규 파일 자동 전환 시 소재 등록자에게 "교체 완료, 신규 파일 반영됨" 알림

### I-23. 매체사 간 컨텍스트 스위칭 비용 — 통합 대시보드 필요 [중]
- **현상**: 브라우저 탭 3개로 3개 매체사 CMS에 각각 로그인. 계정/비밀번호가 각각 다름. 탭 제목이 동일하여 매체사 혼동. 한 매체사 작업 중 다른 매체사 급한 건 처리 후 복귀 시 "어디까지 했더라?" 혼란. 잘못된 매체사에 편성 저장 사고 (R1 I-15와 동일 맥락이나, 여기서는 **계정 분리에 의한 워크플로우 단절**에 초점)
- **문제**: 현재 시나리오에 운영대행사의 다중 매체사 관리 워크플로우가 반영되어 있지 않음. 권한 모델상 운영대행사는 "부여받은 매체"에 접근 가능하나, **하나의 세션에서 복수 매체사의 매체를 관리하는 UX**가 정의되지 않음
- **관련**: 공통 UX, 01 권한 모델
- **제안**: (1) 운영대행사 전용 "크로스 매체사 뷰" — 하나의 로그인으로 부여받은 모든 매체사의 매체/편성을 매체사별 섹션으로 조회, (2) 매체사 전환 드롭다운을 헤더에 상시 표시 (탭 분리 불필요), (3) 편성표 저장 시 소속 매체사명 강조 확인 (I-15 제안과 연계)

### I-24. 소재 검수 실패 시 비전문 사용자를 위한 대응 가이드 부재 [중]
- **현상**: "H.264 코덱이 아닙니다" 오류 메시지를 광고주에게 그대로 전달. 광고주가 "MP4인데요?"라고 반문하면 설명 불가. 영상 제작사에 문의하라고 안내하지만 시간 지연. 이준혁 본인도 H.264가 뭔지 정확히 모름
- **문제**: 03 미결에 이미 "[인터뷰 도출] 검수 실패 사유 문구 친절화 필요"가 있으나, 이는 영업대행사 관점에서 도출된 것. 운영대행사도 동일한 문제를 겪으며, 특히 **직접 소재를 올리는 빈도가 영업대행사보다 높아** 체감 강도가 더 큼. 현재 03 미결의 "수정 가이드 표시" 제안이 구체화되지 않음
- **관련**: 03 UC-13, UC-14, 03 미결 "검수 실패 사유 문구 친절화"
- **제안**: (1) 검수 실패 메시지에 "해결 방법" 섹션 필수 포함 — 예: "H.264 코덱의 MP4 파일만 허용됩니다. → 영상 편집 프로그램에서 H.264로 내보내기 하세요. [상세 가이드 보기]", (2) 상세 가이드는 스크린샷 포함 PDF/웹페이지로 제공, (3) "이 안내를 복사하여 제작사에 전달" 원클릭 복사 버튼 (03 미결 "복사 버튼" 제안과 통합)

### I-25. 편성표 수정 시 "적용중" 상태의 위험성 인지 부족 [하]
- **현상**: 적용중인 편성표를 수정할 때 특별한 경고가 없어, 일반 편성표 수정과 동일하게 진행. "즉시 반영"을 누르면 즉시 현장이 바뀐다는 긴장감이 없음. 실수로 구좌 하나를 삭제한 채 즉시 반영 → 현장에서 해당 구좌 스킵 → 광고주 "왜 우리 광고 빠졌어요?" 클레임
- **문제**: UC-25 "적용중 편성표의 재생목록을 수정" 항목에 즉시 반영 필요성만 기술. **적용중 편성표 수정 진입 시 경고**나 **즉시 반영 전 변경사항 요약 확인**이 시나리오에 없음. 페르소나 디자인 시사점에 "저장 전 변경사항 요약 확인 단계"가 있으나 시나리오에 미반영
- **관련**: 05 UC-25, UC-21a, SCH-BR-10
- **제안**: (1) 적용중 편성표 수정 진입 시 "이 편성표는 현재 매체에서 재생 중입니다. 수정 후 '즉시 반영' 시 즉시 변경됩니다" 경고 배너, (2) "즉시 반영" 클릭 시 변경사항 요약(추가/삭제/순서변경 내역) 확인 모달, (3) 실수 방지를 위한 "되돌리기" 기능 — 직전 반영 상태로 1회 롤백 가능

- **디바이스 ID QR 코드 스캔 가능성**: 플레이어 팀에 QR 코드 표시 기능 구현 가능 여부 확인 필요. 현장 기사의 스마트폰 환경(OS, 카메라 성능) 조사 필요
