# 여정지도 — 이준호 (바리 챗봇)

**페르소나:** 이준호, 3년차 실무 직원
**시나리오:** 처음 처리하는 업무에서 절차를 모를 때 바리 챗봇으로 해결하는 전체 경험
**여정 범위:** 업무 막힘 발생 → 챗봇 사용 → 업무 완료

---

## 여정 단계

### 1단계: 막힘 인식 (Trigger)
**감정: 😣 불안**

| 항목 | 내용 |
|------|------|
| 목표 | 업무를 처리하고 싶은데 뭔가 모름 |
| 행동 | 업무 중 막히는 상황 발생, 포털 검색 시도 |
| 터치포인트 | 사내 포털, 공유 폴더 |
| Pain Point | 검색 결과가 없거나 오래된 문서만 나옴 |
| 기회 | **"사내 문서 검색"으로 바리 챗봇 진입 유도** |

---

### 2단계: 챗봇 발견 (Discovery)
**감정: 😐 중립 → 약간 호기심**

| 항목 | 내용 |
|------|------|
| 목표 | 바리가 뭔지, 도움이 될지 판단 |
| 행동 | 대시보드에서 바리 챗봇 iframe 발견 |
| 터치포인트 | JAVA 대시보드 내 바리 위젯 |
| Pain Point | "이게 뭐지? 진짜 답을 줄 수 있나?" 신뢰 불확실 |
| 기회 | **빈 화면 첫 메시지에서 가능한 것을 명확히 보여줄 것** |

---

### 3단계: 첫 질문 (First Use)
**감정: 😐 어색함 → 기대**

| 항목 | 내용 |
|------|------|
| 목표 | 막힌 것을 물어보고 싶은데 어떻게 써야 할지 모름 |
| 행동 | 예시 pill 클릭 또는 직접 질문 입력 |
| 터치포인트 | 챗봇 입력창, 후속 질문 pill |
| Pain Point | "어떻게 물어봐야 하지?" 첫 질문의 심리적 장벽 |
| 기회 | **초기 화면 예시 질문 pill이 첫 장벽을 낮춤** |

---

### 4단계: 답변 수신 (Response)
**감정: 😊 안도 or 😕 실망 (결정적 순간)**

| 항목 | 내용 |
|------|------|
| 목표 | 명확한 답 + "이게 맞다"는 확신 |
| 행동 | 답변 읽기, 출처 확인, 상세 보기 클릭 |
| 터치포인트 | 답변 버블, 출처 표시, 요약/상세 토글 |
| Pain Point | 답변이 애매하거나 출처가 없으면 신뢰 불가 |
| **Moment of Truth** | **📎 출처가 있는 순간 — 이게 "공식 답변"으로 느껴지는 순간** |
| 기회 | 출처 클릭 가능하게 → 원문 확인 경로 제공 |

---

### 5단계: 심화 탐색 (Deepening)
**감정: 😌 자신감 상승**

| 항목 | 내용 |
|------|------|
| 목표 | 처음 질문에서 파생된 다음 단계도 알고 싶음 |
| 행동 | 후속 질문 pill 클릭 또는 추가 질문 입력 |
| 터치포인트 | 후속 질문 chip, 대화 이어가기 |
| Pain Point | 연관 질문이 없으면 다시 막힘 발생 |
| 기회 | **후속 질문 pill이 "다음 단계"를 자동으로 안내** |

---

### 6단계: 업무 완료 (Completion)
**감정: 😄 성취감 — "유능해졌다" 순간**

| 항목 | 내용 |
|------|------|
| 목표 | 업무를 완료하고 제대로 처리했다는 확신 |
| 행동 | 챗봇에서 얻은 정보로 업무 완료 |
| 터치포인트 | 실제 업무 시스템 (챗봇 외부) |
| Pain Point | 챗봇 답변과 실제 현실이 다를 경우 신뢰 훼손 |
| 기회 | **최신 문서 반영이 신뢰의 근거 — 관리자 역할이 중요** |

---

## 감정 곡선

```
😣      😐      😐      😊      😌      😄
막힘  → 발견  → 첫질문 → 답변  → 심화  → 완료
  ↓               ↑
  (출처 없으면 😕 로 급락)
```

---

## 핵심 발견 (Moments of Truth)

| 순위 | 순간 | 임팩트 |
|------|------|--------|
| 1 | 출처 표시된 답변을 받는 순간 | "이게 공식 답이다" 확신 → 유능함 느낌의 근원 |
| 2 | 후속 질문 pill로 다음 단계까지 해결되는 순간 | "나 혼자 다 해결했다" 성취감 |
| 3 | 첫 질문 pill로 시작이 쉬운 순간 | 진입 장벽 해소 |

---

## 상위 3개 디자인 기회

1. **출처의 시각적 강조** — 답변보다 출처가 더 눈에 띄어야 신뢰감 즉각 전달
2. **후속 질문의 맥락 연결** — 단순 관련 질문이 아니라 "다음 단계" 흐름으로 설계
3. **첫 화면 온보딩** — 빈 DB든 아니든, 바리가 어떤 도움이 되는지 즉각 보여주기
