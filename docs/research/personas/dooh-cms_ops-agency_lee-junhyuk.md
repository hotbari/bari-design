# 이준혁 (운영대행사)

## 기본 정보
- **이름**: 이준혁
- **나이**: 29세
- **직책**: 옥외광고 운영대행사 "애드옵스" 주니어 매니저
- **역할**: 운영대행사
- **한 줄 요약**: "이거 제가 잘못 건드리면 큰일나는 거 아니에요?"

## 인구통계
- 기술적 이해도: **낮음** (UI만 사용, 문제 시 매체사/어드민에 연락)
- 3개 매체사의 매체 총 150대 위탁 관리 중

## 목표
- **기능적**: 여러 매체사 편성을 실수 없이 동시에 관리
- **감정적**: "내가 저장한 편성이 정확히 반영되었다"는 안심감
- **사회적**: 매체사로부터 "일 잘한다" 신뢰, 광고주 클레임 제로

## 고충 (Frustrations)
- 여러 매체사 편성을 동시에 관리하다 실수하기 쉬움
- 편성 충돌 시 어떤 편성이 우선인지 헷갈림
- 다른 사용자와 동시 편성 수정 시 덮어쓰기 사고 (30분 작업 날아감)
- 급한 소재 교체 요청이 검수에 걸리면 시간 압박
- 편성 적용 후 실제 플레이어 재생 상태 확인 불가

## 행동 패턴
- CMS에서 편성표 생성, 재생목록 관리, 소재 업로드 매일 수행
- 브라우저 탭 여러 개 띄워놓고 3개 매체사 동시 작업
- 편성 충돌 발생 시 매체사에 확인 요청 (본인 판단 회피)
- 급한 건은 매체사 담당자에게 직접 전화

## 시나리오 (Day-in-the-Life)
오전 9시, 매체사 A의 주말 특별 편성 작성 시작. 동시에 매체사 B의 광고주가 "오늘 오후부터 소재 바꿔달라" 요청. 소재 올리고 검수 대기하며 매체사 A 편성 계속 작업. 30분 후 저장했는데 — 매체사 A 담당자가 같은 편성을 수정 중이었고, 본인 작업이 덮어써짐. 월요일 아침 광고주 B "왜 우리 광고 안 나갔어요?" 전화.

## 디자인 시사점
- 편성표 동시 수정 시 경고 (낙관적 잠금 + 충돌 안내)
- 편성 충돌 해소 UI에서 우선순위 규칙 명확히 안내
- 소재 검수 상태·예상 완료 시간 실시간 표시
- 작업 실수 방지: 저장 전 변경사항 요약 확인 단계
- 편성 적용 후 플레이어 동기화 상태 피드백
