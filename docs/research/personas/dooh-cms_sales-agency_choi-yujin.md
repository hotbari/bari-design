# 최유진 (영업대행사)

## 기본 정보
- **이름**: 최유진
- **나이**: 31세
- **직책**: 광고대행사 "브라이트미디어" 영업팀
- **역할**: 영업대행사
- **한 줄 요약**: "광고주가 내일 온에어래요, 검수 언제 끝나요?"

## 인구통계
- 기술적 이해도: **낮음** (이미지/영상 파일 업로드, 상태 확인만)
- 5개 광고주의 옥외광고 소재 관리
- CMS 접근 범위: 소재 도메인만 (편성, 매체 관리 접근 불가)

## 목표
- **기능적**: 소재 업로드 → 검수 통과 → 송출 확인까지 빠르게
- **감정적**: "내 소재 상태가 뭔지, 문제면 뭐가 문제인지, 언제까지 되는지" 알 수 있는 안심감
- **사회적**: 광고주에게 정확한 일정 안내, 신뢰 유지

## 고충 (Frustrations)
- 소재 검수가 언제 끝나는지 모름 ("검수중"에서 영구 정체 경험)
- 검수 실패 시 사유가 불명확 → 수정 방향을 모름
- 광고주의 "내일 온에어" 압박 vs 검수 대기 시간 갭
- 매체 오프라인 등 송출 문제 발생 시 알림을 받지 못함 → 광고주에게 답변 불가
- 같은 파일인데 검수 결과가 달라지는 비일관성

## 행동 패턴
- 광고주로부터 최종 소재 수령 → CMS에 즉시 업로드
- 검수 상태 수시 확인 (새로고침 반복)
- 검수 실패 시 매체사 담당자에게 전화로 문의
- 광고주에게 "정상 반영됐습니다" 보고 후 실제 송출은 확인 불가

## 시나리오 (Day-in-the-Life)
오후 2시, 광고주 "프레시주스" 신제품 론칭 소재 최종본 수령. 내일 오전 6시 온에어. 바로 CMS에 업로드. "검수중" 상태. 4시가 되도 안 끝남. 매체사에 전화 — "시스템 보겠다"만 답변. 저녁 8시, 불안해서 같은 파일 재업로드. 원래 건은 "검수 실패 — 규격 불일치", 재업로드 건은 "검수 통과". 같은 파일인데. 광고주에겐 "정상 처리됐습니다" 보고했지만, 시스템 신뢰가 깨짐.

## 디자인 시사점
- 검수 예상 소요 시간 표시 + 타임아웃 시 명확한 안내
- 검수 실패 사유를 비기술적 언어로 구체적 표시 (예: "해상도 불일치: 1920×1080 필요, 1280×720 업로드됨")
- 소재 상태 타임라인 (업로드 → 검수중 → 완료, 각 단계 소요 시간 표시)
- 내 소재가 송출되는 매체에 문제 발생 시 최소한의 알림 제공
- 검수 통과 기준을 업로드 전에 미리 안내 (파일 규격 가이드)

## 인터뷰 도출 (2026-04-05, 2차 심화)

### 검증된 가설
- 검수 실패 수정 가이드 필요성 강하게 검증 — AE→광고주→제작사 3단 전달 문제 확인. 캡쳐 시 타 광고주 정보 노출 위험도 추가 확인
- 매체 오프라인 알림 영업대행사 수신 필요성 강하게 검증 — "럭키카페" 광고주가 종로 전광판 현장에서 먼저 장애 인지한 구체 사례 확인
- 업로드 전 매체별 규격 표시가 검수 실패율 감소에 유효함 검증 — 제작사에 규격 전달 시 엑셀 수동 정리 우회 패턴 확인
- 편성/매체 메뉴 숨김(완전 비표시) 방식이 UX상 적절함 검증 — "잘못 누를 일 없어 편하다" 직접 응답

### 기존 발견 (심화)
1. **검수 실패 가이드 복사 버튼** — 캡쳐→카톡 전달 워크플로우 확인 + 캡쳐 시 타 광고주 소재 정보 노출 보안 위험 추가 확인. "공유용 클린 텍스트" 생성 필요 [소재 관리, 중→상]
2. **소재 상세 내 송출 매체 목록 읽기 전용 뷰** — "강남역 전광판에 나오는 거 맞죠?" 질문 대응 불가 사례 확인. 나아가 "검수완료 ≠ 송출중" 상태 갭 문제까지 확장 — 편성 연결 상태(미편성/편성대기/송출중/송출종료) 표시 필요 [소재/편성 교차, 상]
3. **매체 오프라인 알림 수신 대상에 영업대행사 추가** — 해당 매체에 소재 편성된 영업대행사도 포함. "미리 알면 선제 안내 가능, 그게 신뢰 관리" 직접 인용 [알림 관리, 상]
4. **운영 기간 만료 알림 2단계화** — 7일 전도 빠듯할 수 있음. 10일 전 1차 → 3일 전 2차로 확장 검토 (광고주 의사결정 + 제작 + 검수 리드타임) [알림 관리, 중]
5. **소재 공유 설정 주체 재검토** — "다른 대행사? 절대 안 돼요" 명확한 반응. 기본 비공개, 광고주 명시적 승인 시에만 공유 [소재 관리, 중]
6. **매체별 소재 규격 안내 독립 페이지** — 현재 엑셀로 정리해 제작사에 전달하는 우회. 링크 공유 가능한 규격 가이드 요구 [소재 관리, 중]

### 신규 발견 (2차)
7. **검수완료↔송출중 상태 갭 — 광고주 오보고** — 검수완료 시점에 "반영됐습니다" 보고하나 실제 편성 전이면 미송출. 소재 도메인 내 편성 연결 상태 표시 필요 [소재/편성 교차, 상]
8. **수동검수대기 전환 인지 실패** — 인앱 알림만으로는 AE에 도달 못함. 금요일 전환 시 주말 방치. 알림 우선순위 상향 + 모바일 채널 필요 [알림 관리, 상]
9. **매체 선택 오류 미감지** — 일괄 업로드 시 매체 잘못 선택해도 검수 통과(메타데이터 기준 검증). 매체 실제 해상도와 교차 검증 항목 UC-14에 추가 필요 [소재 관리, 중]
10. **소재 교체 이력 미보존** — 광고주 "이전 버전으로 돌려주세요" 요청 시 대응 불가. 로컬 PC 수동 보관 우회. 03 미결 "버전 관리 필요 여부" 실무 수요 확인됨 [소재 관리, 중]
11. **영업대행사용 소재 현황 대시보드 부재** — 40~50개 소재 상태 파악에 일일이 클릭 필요. 상태별 요약(검수중/완료/실패/만료임박) 한눈에 보는 대시보드 요구 [대시보드, 중]
12. **모바일 알림 채널 부재** — 외근 빈번한 AE에게 인앱/브라우저 푸시/Slack 모두 부적합. SMS 또는 카카오 알림톡 채널 추가 검토 필요 [알림 관리, 상]

### 페르소나 정의 보정
- 접근 범위 인식: "메뉴 숨김은 편하지만 소재 송출 현황은 소재 도메인 내에서 보고 싶다" — 접근 범위를 메뉴 단위가 아닌 데이터 조회 단위로 세분화 필요
- 만료 알림 활용: 3일 전 알림만으로 부족, 7~10일 전 사전 안내 → 광고주 연장/교체 의사 확인 리드타임 필요
- 소재 공유: 필요하되 기본 비공개, 광고주 승인 시에만 공유 허용
- 알림 채널 선호: 인앱 < 브라우저 푸시 < 카카오톡/SMS 순. CMS 비상주 패턴 반영 필요
- CMS 사용 패턴 보정: "종일 상주"가 아닌 "오후 간헐적 접속" — 외근·미팅 사이 틈새 확인 패턴
