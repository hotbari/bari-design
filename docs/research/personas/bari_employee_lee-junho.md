# 이준호 — 3년차 실무 직원 (Primary Persona)

> "찾는 데 시간 다 쓰고 정작 일을 못 해."

## Demographics
- 나이: 28세
- 직무: 기획/운영팀 3년차
- 기술 수준: 스마트폰·업무툴 익숙, 사내 시스템 무난
- 업무 패턴: 혼자 처리해야 하는 상황이 많음

## Goals
- **Functional:** 모르는 절차를 선배 도움 없이 빠르게 해결
- **Emotional:** 스스로 해결했다는 성취감
- **Social:** 팀에서 "일 잘 처리하는 사람"으로 인정받기

## Frustrations
- 규정 문서가 어디 있는지 매번 찾아야 함
- 어떤 버전이 최신인지 모름
- 선배에게 물어보면 "그런 것도 몰라?" 하는 눈치
- 키워드가 안 맞으면 검색 결과가 아무것도 안 나옴

## Behaviors
- 동료/선배에게 먼저 물어봄 → 없으면 사내 포털 검색
- 관련 없는 문서 3~4개 열어보다 포기하는 경우 多
- 확실하지 않은 채로 업무 처리 후 나중에 수정

## Day-in-the-Life Scenario
신규 거래처 계약을 처음 처리하는 월요일 오전. 팀장은 회의 중, 담당 선배는 재택. 포털에서 "거래처 등록"을 검색하면 2020년 문서가 나온다. 어느 양식이 맞는지 모르겠고, 전화하기도 눈치 보인다. 결국 예전 계약서 파일을 참고해서 작성하고, 나중에 틀렸다는 걸 알게 된다.

## Design Implications
- 절차형 질문에 단계별 명확한 답변 필수
- 출처 표시로 "이게 공식 답변이다" 신뢰감 제공
- 후속 질문 제안으로 혼자 해결 완주 경험 만들기
- 짧고 요약된 답변 우선, 필요할 때 상세 보기
