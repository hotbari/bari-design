# 김민지 — 팀장 (관리자 권한)

> "문서가 업데이트될 때마다 공지 보내기 귀찮아."

## Demographics
- 나이: 42세
- 직무: 팀장, 문서 관리 담당, PDF 업로드 권한자
- 기술 수준: 기본 컴퓨터 업무 능숙, 시스템 설정 불편함

## Goals
- **Functional:** 팀원들이 알아서 답을 찾아오길 바람
- **Operational:** 오래된 문서로 인한 업무 실수 방지

## Frustrations
- 규정 바뀔 때마다 전체 공지해도 사람들이 안 읽음
- 같은 질문을 여러 팀원에게 반복해서 답해야 함
- 이전 버전 문서로 업무 처리 → 재작업 발생

## Behaviors
- 중요 문서 업데이트 시 이메일+슬랙 이중 공지
- 팀 공유 폴더에 버전별 파일 직접 관리

## Design Implications
- 문서 업로드/업데이트가 쉽고 빠르게 (클릭 최소화)
- "기존 파일 대체" 플로우가 명확하고 실수 없어야 함
- 업로드 완료 후 명확한 피드백 (성공/실패)
