# 김서영 (매체사)

## 기본 정보
- **이름**: 김서영
- **나이**: 34세
- **직책**: 옥외광고 매체사 "시티미디어" 운영팀장
- **역할**: 매체사
- **한 줄 요약**: "현장에서 진짜 이렇게 돌아가나요?"

## 인구통계
- 기술적 이해도: **중간** (UI로 대부분 처리, API는 모름)
- 서울·수도권 옥외 광고판(빌딩 외벽 전광판, 도로변 디지털 사이니지 등) 운영
- 운영대행사 2곳, 영업대행사 3곳에 권한 부여 중

## 목표
- **기능적**: 광고주 요청에 맞춰 편성 변경이 제때 현장에 반영되는 것
- **감정적**: "CMS에서 보이는 상태가 현장과 같다"는 확신
- **사회적**: 광고주에게 "네, 정상 송출 중입니다" 자신 있게 답변

## 고충 (Frustrations)
- 편성 변경 후 실제 현장 반영 여부 원격 확인 어려움
- 플레이어 오프라인 복구 후 구버전 편성이 나가는 사고 (정산 분쟁)
- 급한 소재 교체 시 검수 대기 시간 압박
- 여러 대행사가 동시에 편성 작업할 때 덮어쓰기 사고

## 행동 패턴
- CMS에서 매체 등록, 소재 검수 확인, 편성표 관리 직접 수행
- 광고주 요청 시 편성표 수정 후 "반영 완료" 확인까지 체크
- 문제 시 운영대행사 또는 어드민에게 에스컬레이션
- 월말 정산 시 송출 리포트와 계약 대조

## 시나리오 (Day-in-the-Life)
오전 10시, 광고주 A로부터 "이번 주 금요일부터 소재 바꿔주세요" 요청 메일. 영업대행사에 소재 업로드 요청 → 검수 대기. 오후, 운영대행사가 편성표 수정 완료 보고. CMS에서 "적용중" 확인했지만, 실제 현장 반영 여부는 알 수 없어 불안. 다음날 광고주가 "확인했는데 옛날 소재 나오고 있다" 연락 — 플레이어 30대가 네트워크 이슈로 재동기화 안 된 것. 무상 집행 보상 처리.

## 디자인 시사점
- 편성 반영 상태를 매체별로 "의도 vs 실제" 비교 표시
- 플레이어 동기화 미완료 매체 목록 한눈에 확인
- 소재 교체 시 검수 진행률·예상 소요 시간 표시
- 편성 수정 이력 (누가, 언제, 뭘 바꿨는지) 명확히 추적
