## Design Context

### Users
- **어드민** (박진수): IT 운영 10년차, 기술적 이해도 높음. 시스템 안정성과 장애 감지가 최우선. 대시보드에서 이상 징후를 빠르게 파악해야 함
- **매체사** (김서영): 운영팀장, 중간 기술 이해도. 여러 매체의 편성/캠페인/소재를 동시 관리. 엑셀 병행 관리를 CMS로 대체하고 싶어 함
- **운영대행사** (이준혁): 주니어 매니저, 기술 이해도 낮음. 3개 매체사 150대 매체를 동시 관리. 실수 공포가 크고, 판단을 매체사에 위임하는 패턴
- **영업대행사** (최유진): AE, 기술 이해도 낮음. 소재 도메인만 접근. 광고주와의 커뮤니케이션 브릿지. CMS에 상주하지 않음

**사용 맥락**: B2B 운영 도구. 매일 반복 사용하는 업무 시스템. 긴급 상황(매체 장애, 긴급 편성) 대응도 필요. 디지털에 익숙하지 않은 현장 인력도 사용.

### Brand Personality
- **3단어**: 신뢰 · 편이 · 깔끔
- **감정 목표**: "이 시스템은 내가 실수하지 않게 도와준다"는 안심감. 복잡한 운영을 단순하게 느끼게 하는 명료함
- **톤**: 네이버 계열 — 인프런처럼 깔끔하고 친근하되 전문적. 과하지 않은 초록 계열 포인트
- **절대 아닌 것**: AI Slop 양산형 (그라디언트 텍스트, 글래스모피즘, 다크모드+글로잉 액센트, 제네릭 히어로 카드). 화려함보다 명확함

### Aesthetic Direction
- **비주얼 톤**: 미니멀 + 데이터 밀도 적절 + 가시성 우선
- **테마**: 라이트 모드 전용 (다크 모드 추후)
- **색상 방향**: 네이버 톤 유지 — 초록 계열 Primary, 차분한 뉴트럴. AI Slop 색상(인디고-퍼플 그라디언트, 네온 액센트) 배제
- **타이포그래피**: 한글 가독성 우선. 본문은 Pretendard 또는 Noto Sans KR 검토
- **레퍼런스 방향**: 인프런의 깔끔함 + 운영 도구의 정보 밀도
- **안티 레퍼런스**: 제네릭 SaaS 랜딩, 글래스모피즘 대시보드, 그라디언트 카드 UI

### Design Principles
1. **명확함 우선 (Clarity First)**: 모든 상태, 액션, 결과가 한눈에 파악 가능해야 한다. 장식보다 정보 전달
2. **실수 방지 (Error Prevention)**: 파괴적 액션은 확인 단계 필수. 현재 상태와 결과를 예측 가능하게 표시. "되돌릴 수 없는 작업"은 명시적 경고
3. **유저 유도 (Guided Flow)**: 다음 단계가 자연스럽게 보여야 한다. 복잡한 워크플로우는 단계 표시. 빈 상태에서도 "다음에 할 일"을 안내
4. **역할별 적정 노출 (Role-Appropriate Exposure)**: 각 역할이 필요한 정보만 본다. 권한 밖 기능은 완전 숨김. 알림도 역할에 맞게 필터링
5. **현장 신뢰 (Field Trust)**: CMS 상태와 현장 실제가 일치해야 한다. 불일치 시 즉시 시각적 표시. 동기화 상태를 항상 확인 가능
