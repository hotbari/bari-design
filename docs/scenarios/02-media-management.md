# 매체 관리 시나리오

## 1. 개요

CMS에서 매체사(회사)와 매체(디스플레이 장비)를 등록·설정·모니터링한다. 매체사 등록은 어드민 전용이며, 매체 등록은 매체사와 운영대행사도 가능하다. 매체-디바이스 연동, SSP 연동, 유동인구 데이터 연결까지 포함한다.

## 4. 상세 시나리오

### UC-06: 어드민이 매체사 등록

**누가**: 어드민
**언제**: 신규 매체사를 CMS에 온보딩할 때
**전제조건**: 어드민이 로그인된 상태

#### 시나리오

1. 어드민이 매체사 관리 > 등록 버튼을 클릭한다
2. 필수 정보를 입력한다:
   - 사업자등록번호
   - 회사명
   - 대표자명
   - 담당자 연락처 (이메일, 전화번호)
3. 선택 정보를 입력한다:
   - 회사 주소
   - 비고 (메모)
4. 등록 버튼을 클릭한다
5. 시스템이 매체사 식별 ID를 자동 발급한다
6. 매체사 관리 목록에 신규 매체사가 표시된다

#### 이런 경우에는

- **사업자등록번호 중복**: "이미 등록된 매체사입니다" 안내 → 기존 매체사 상세로 이동 링크 표시
- **필수 항목 누락**: 해당 필드 하이라이트 + 안내 메시지. 등록 버튼 비활성화
- **사업자등록번호 형식 오류**: "올바른 사업자등록번호 형식(000-00-00000)을 입력해주세요" 안내

#### 결과

- 매체사 레코드 생성: 상태 "등록됨", 매체사 식별 ID 자동 발급
- 이후 UC-01(초대)을 통해 매체사 담당자 계정 생성 가능
- 등록 완료 후 '담당자를 바로 초대하시겠습니까?' CTA 표시. 미초대 매체사(등록 후 7일 경과 + 초대 0건)에 대해 어드민에게 리마인더 알림 발송

### UC-07: 매체사가 매체 등록

**누가**: 매체사 또는 운영대행사
**언제**: 새로운 디스플레이 장비를 CMS에 등록할 때
**전제조건**: 매체사 사용자가 로그인된 상태. 소속 매체사가 UC-06으로 등록 완료

#### 시나리오

1. 매체사가 매체 관리 > 등록 버튼을 클릭한다
2. **기본 정보**를 입력한다:
   - 매체명 (필수)
   - 매체 종류: 고정형 / 이동형 (필수)
   - 설치 주소 (필수. 이동형의 경우 거점 주소 + 도움말 표시)
   - 설치 형태: 가로형 / 세로형 (필수)
3. **디스플레이 설정**을 입력한다:
   - 디스플레이 유형: LCD / LED / 기타(직접입력) (필수)
   - 화면 사이즈 (필수)
   - 해상도 (필수)
   - 허용 해상도 (필수)
   - LED인 경우: 비정형 사이즈 여부, pitch 값 (네이버 RFP 필수 항목)
4. **운영 시간**을 설정한다:
   - 요일: 매일 / 평일 / 주말 / 직접 선택
   - 시간: 24시간 / 직접 입력 (시작~종료)
   - 목록 표시 형식: `xx:xx ~ xx:xx`
5. **비고**를 입력한다 (선택)
6. 등록 버튼을 클릭한다
7. 시스템이 매체 식별 코드(Media Code)를 자동 발급한다
8. 시스템이 플레이어 연결 시점을 자동 생성한다 (타 매체와 겹치지 않게 분산)

#### 이런 경우에는

- **운영대행사가 권한 범위 밖 매체 등록 시도**: 등록 버튼이 표시되지 않음 (권한 필터링)
- **LED 선택 시 pitch 값 미입력**: "LED 매체는 pitch 값을 필수로 입력해야 합니다" 안내
- **운영 시간 시작 > 종료**: "시작 시간이 종료 시간보다 앞서야 합니다" 안내
- **이동형 매체 선택 시**: 설치 주소 라벨이 "거점 주소"로 변경, 도움말 툴팁 표시
- **매체명 중복 (동일 매체사 내)**: "동일한 이름의 매체가 이미 존재합니다" 안내

#### 결과

- 매체 레코드 생성: 상태 "미연동" (디바이스 연동 전)
- 매체 식별 코드(Media Code) 자동 발급 — 매체 관리용 시스템 ID. 디바이스 ID(물리 장비 식별자)와는 별개
- 플레이어 연결 시점 자동 생성 (수정은 어드민만 가능)

### UC-08: 매체-디바이스 연동

**누가**: 매체사 또는 운영대행사
**언제**: 현장에 플레이어를 설치한 후, CMS 매체와 물리 장비를 연결할 때
**전제조건**: UC-07로 매체 등록 완료. 현장에 플레이어 소프트웨어 설치 완료. 디바이스 ID는 현장 플레이어에 표시되는 물리 장비 식별자 (시스템 발급이 아닌 장비 고유값)

#### 시나리오

1. 매체사가 매체 상세 > 디바이스 연동 탭을 클릭한다
2. 현장 플레이어에 표시된 디바이스 ID를 입력한다 (수동 입력 또는 QR 코드 스캔)
3. 연동 버튼을 클릭한다
4. 시스템이 플레이어와 통신을 시도한다
5. 연동 성공 시 매체 상태가 "미연동" → "연동됨"으로 변경된다
6. **영상 출력 조정** 단계로 진입한다:
   - 플레이어에서 디바이스 영상 사이즈와 위치를 방향키로 육안 조정
   - 조정값이 실시간으로 CMS에 동기화
7. 조정 완료 후 "잠금" 버튼을 클릭한다
8. 크기/위치가 고정되어 오조작을 방지한다

#### 이런 경우에는

- **잘못된 디바이스 ID 입력**: "유효하지 않은 디바이스 ID입니다" 안내
- **이미 다른 매체에 연동된 디바이스 ID**: "이 디바이스는 다른 매체(매체명)에 연동되어 있습니다" 안내. 연동 차단
- **플레이어 통신 실패**: "플레이어와 연결할 수 없습니다. 네트워크 상태를 확인해주세요" 안내. 재시도 버튼 표시
- **잠금 해제 필요**: CMS 또는 플레이어에서 "잠금 해제" 클릭 → 재조정 가능
- **잠금 상태에서 사이즈/위치 변경 시도**: 변경 불가. "잠금을 해제해주세요" 안내
- **연동 성공 후 확인 절차**: 시스템이 '이 장비가 맞습니까? 설치 위치: [매체 주소]' 확인 다이얼로그를 표시한다. 확인 시 테스트 이미지를 송출하여 현장에서 육안 확인 가능
- **조정 완료 미잠금**: 조정 완료 후 5분 이내 잠금하지 않으면 '잠금을 잊으셨나요?' 안내 표시. 자동 잠금 옵션 설정 가능 (어드민)
- **디바이스 교체 (장비 고장 등)**: 매체 상세 > 디바이스 연동 탭에서 "디바이스 교체" 버튼 클릭 → 기존 연동 해제 확인 모달: "기존 디바이스 연동을 해제하고 새 디바이스를 연동합니다. 기존 편성표는 유지됩니다. 계속하시겠습니까?" → 확인 시 기존 디바이스 ID 해제 → 새 디바이스 ID 입력 → 연동 테스트 → 성공 시 기존 편성표의 재생목록을 새 장비에 자동 전송
- **디바이스 교체 시 연동 테스트 실패**: 기존 디바이스 ID는 이미 해제된 상태이므로 매체 상태가 "미연동"으로 전이. "새 디바이스 연동에 실패했습니다. 매체가 미연동 상태입니다. 재시도하거나 다른 디바이스 ID를 입력해주세요" 안내. 기존 편성표는 "일시정지" 상태로 전환 (송출 불가). 재연동 성공 시 편성표 자동 복귀

#### 결과

- 매체 상태: "미연동" → "연동됨"
- 영상 출력 사이즈/위치 값 CMS 저장
- 잠금 설정 시 크기/위치 고정
- 디바이스 교체 시: 기존 편성표 유지, 새 장비로 재생목록 자동 재전송

### UC-09: SSP 연동 설정

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 매체를 외부 SSP(Supply-Side Platform)와 연동할 때
**전제조건**: UC-07로 매체 등록 완료. SSP 연동은 필수가 아닌 선택 항목

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 매체 상세 > SSP 연동 탭을 클릭한다
2. SSP를 선택한다 (1차: 네이버, 이후 확장 예정)
3. SSP별 연동 정보를 입력한다:
   - 네이버: 매체 매핑 ID, 보장 인벤토리, 최소 보장단가
4. 연동 테스트 버튼을 클릭한다
5. 시스템이 SSP와 통신 테스트를 수행한다
6. 테스트 성공 시 연동 상태가 "연동 완료"로 표시된다
7. 저장 버튼을 클릭한다

#### 이런 경우에는

- **SSP 통신 테스트 실패**: "SSP 연동 테스트에 실패했습니다. 입력 정보를 확인해주세요" 안내. 저장은 가능하되 연동 상태 "미연동"
- **보장 인벤토리/단가 미입력**: 경고 표시. 저장은 가능하나 네이버 연동 시 필수
- **이미 연동된 SSP를 재설정**: 기존 설정 표시 → 수정 → 저장 시 덮어쓰기

#### 결과

- SSP 연동 정보 저장
- 연동 테스트 성공 시: SSP 연동 상태 "연동 완료"
- 네이버 SSP 연동 시: 편성 관리에서 네이버 연동 구좌 사용 가능

### UC-10: 유동인구 데이터 연결

**누가**: 어드민
**언제**: 매체 위치의 유동인구 데이터를 연결하여 광고 가치를 산정할 때
**전제조건**: UC-07로 매체 등록 완료. SKP FTP에서 유동인구 데이터 수신 가능 상태. 필수가 아닌 선택 항목 (네이버 RFP 관련 항목)

#### 시나리오

1. 어드민이 매체 상세 > 유동인구 탭을 클릭한다
2. 데이터 소스를 확인한다: SKP FTP → 포도 가공 → CMS 업로드 경로
3. 매체의 설치 주소와 유동인구 데이터 지점을 매핑한다
4. 연결 버튼을 클릭한다
5. 시스템이 해당 지점의 유동인구 데이터를 주기적으로 수신하기 시작한다

#### 이런 경우에는

- **설치 주소와 매핑 가능한 유동인구 지점이 없음**: "해당 위치의 유동인구 데이터가 없습니다" 안내
- **데이터 수신 오류**: "유동인구 데이터 수신에 실패했습니다" 안내 + 오류 상세 로그
- **이동형 매체**: 거점 주소 기준으로 매핑. "이동형 매체는 거점 주소 기준으로 연결됩니다" 안내

#### 결과

- 매체에 유동인구 데이터 연결
- 주기적 데이터 수신 시작
- 대시보드/리포트에서 유동인구 데이터 조회 가능

### UC-11: 매체 헬스체크 모니터링

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 매체의 운영 상태를 확인하거나, 이상 징후 알림을 수신했을 때
**전제조건**: UC-08로 디바이스 연동 완료. 매체 상태 "연동됨"

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 매체 관리 > 목록에서 매체를 선택한다
2. 매체 상세 > 모니터링 탭을 클릭한다
3. 현재 상태가 표시된다:
   - 연결 상태: 온라인 / 오프라인
   - CPU 온도
   - GPU 온도
   - 최근 통신 시점
   - 플레이어 리프레시 상태
4. **플레이어 자동 업데이트**: 콜타임(새벽 ~3시) 통신 시 편성표 수신 및 소재 다운로드뿐 아니라 플레이어 소프트웨어 업데이트도 수행한다. 매체사 실무자에게 사전 고지 병행 (거부감 고려)
5. 이상 징후 발생 시 자동 알림이 발송된다 (UC-26 연계)

#### 이런 경우에는

- **매체 오프라인 감지**: 알림 발송 (인앱 + 브라우저 푸시 + Slack). 목록에서 상태 아이콘 변경
- **CPU/GPU 온도 임계치 초과**: 알림 발송 (인앱 + Slack). "매체 이상 감지" 이벤트
- **오프라인 매체가 복구됨**: "매체 복구" 알림 발송 (인앱)
- **플레이어 오프라인 복구 후 재동기화**: 오프라인 동안 편성이 변경된 경우, 복구 시 최신 편성표를 자동 재전송한다
  - 플레이어가 온라인 복귀 시, CMS에 현재 재생 중인 편성 버전을 보고한다
  - CMS는 보고된 버전과 최신 버전을 비교하여, 불일치 시 최신 재생목록을 즉시 전송한다
  - 전송 실패 시 지수 백오프로 재시도 (최대 3회)
  - 재동기화 완료 시 "편성 재동기화 완료" 알림 발송
- **플레이어 리프레시 실패**: 재구동 상태 및 오류 로그 표시

#### 결과

- 매체 상태 실시간 표시
- 이상 징후 시 알림 이벤트 발생 (알림 관리 도메인 연계)

### UC-12: 매체 활성/비활성 전환

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 매체 운영을 일시 중지하거나 재개할 때
**전제조건**: UC-07로 매체 등록 완료

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 매체 관리 > 목록에서 대상 매체를 선택한다
2. 매체 상세에서 "비활성화" 버튼을 클릭한다
3. 시스템이 확인 모달을 표시한다: "이 매체를 비활성화하시겠습니까? 적용중 편성표 N건이 일시정지됩니다. 예약됨 편성표 M건은 시작일에 자동 일시정지됩니다. 편성표 종료일이 경과하면 자동 종료됩니다."
4. 확인 버튼을 누르면 매체 상태가 "연동됨(비활성)"으로 변경된다

#### 이런 경우에는

- **진행 중인 편성표가 있는 매체 비활성화**: "이 매체에 적용중인 편성표가 N건 있습니다. 비활성화하면 모두 일시정지됩니다. 종료일이 경과한 편성표는 자동 종료됩니다. 계속하시겠습니까?" 경고
- **진행 중인 캠페인이 연결된 매체 비활성화**: "이 매체에 집행 중인 캠페인이 N건 있습니다." 추가 경고
- **비활성 매체를 다시 활성화**: "활성화" 버튼 클릭 → 즉시 활성화. `일시정지` 상태의 편성표 중 종료일이 경과하지 않은 것은 자동으로 `적용중`으로 복귀. 종료일이 이미 경과한 편성표는 `종료` 처리 (재편성 필요)
- **매체 비활성화 시 편성표 상태 경합**: 비활성화/활성화와 편성표 상태 전이를 원자적으로 처리한다
  - 비활성화: 해당 매체의 모든 "적용중" 편성표를 "일시정지"로 전환 (단일 트랜잭션)
  - 활성화: 현재 시각 기준으로 종료일을 재검사한 후 상태 전환 (종료일 경과 → "종료", 미경과 → "적용중")
  - 비활성화/활성화 처리 중 동일 매체에 대한 편성표 수정 요청은 대기 (낙관적 잠금)
- **매체 비활성화 시 캠페인 상태 불일치**: 비활성화 시 해당 매체를 대상으로 하는 "집행중" 캠페인에 대해 알림을 발송한다
  - 대상 매체가 **모두** 비활성화되면: 캠페인 담당자에게 "모든 대상 매체 비활성 — 실제 송출 불가" 경고 알림 발송
  - 대상 매체 중 **일부만** 비활성화되면: "N개 매체 비활성 — 해당 매체 송출 중단" 정보 알림 발송
  - 캠페인 상태 자체는 "집행중" 유지 (편성표 도메인에서 일시정지 처리)
- **운영대행사가 권한 범위 밖 매체 전환 시도**: 버튼이 표시되지 않음

#### 결과

- 매체 상태 변경: "연동됨(활성)" ↔ "연동됨(비활성)"
- 비활성화 시: 해당 매체의 `적용중` 편성표를 `일시정지`로 전환, 광고 송출 중지. **비활성화 주체 ≠ 편성표 생성자인 경우** 편성표 생성자에게 "편성 N건이 일시정지됩니다" 알림 발송
- 활성화 시: `일시정지` 편성표 중 종료일 미경과 → `적용중` 자동 복귀. 종료일 경과 → `종료` 처리

### UC-37: 매체 그룹 관리

**누가**: 어드민 또는 매체사
**언제**: 여러 매체를 그룹으로 묶어 캠페인·편성에서 일괄 선택할 때
**전제조건**: 로그인 상태. 매체가 2개 이상 등록됨

#### 시나리오

1. 어드민 또는 매체사가 매체 관리 > 매체 그룹 > 새 그룹 버튼을 클릭한다
2. 그룹명을 입력한다 (필수)
3. 소속 매체를 선택한다 (복수 선택, 최소 1개)
4. 저장 버튼을 클릭한다

#### 이런 경우에는

- **그룹명 중복 (동일 매체사 내)**: "동일한 이름의 그룹이 이미 존재합니다" 안내
- **그룹 수정**: 그룹 상세에서 소속 매체 추가/제거 가능
- **그룹 삭제**: 확인 모달 후 soft delete (목록 비표시). 캠페인·편성표에서 이 그룹을 참조하던 항목은 개별 매체 참조로 풀어짐 (그룹 해체)
- **보관됨 매체가 그룹에 포함**: 그룹 목록에서 보관됨 매체는 흐리게 표시. 그룹 선택 시 보관됨 매체는 자동 제외
- **운영대행사가 그룹 생성 시도**: 권한 없음. 그룹 관리 메뉴 미표시
- **긴급 편성(UC-35) 시**: 매체 그룹 프리셋으로 원클릭 대상 선택 가능

#### 결과

- 매체 그룹 레코드 생성
- 캠페인 등록(UC-17), 편성표 생성(UC-22)에서 매체 그룹 단위 선택 가능

## 5. 도메인 규칙

### 5.1 매체사 상태 FSM

```
[등록됨] ──(삭제)──→ [삭제됨]
```

매체사는 별도 활성/비활성 상태 없음. 등록 후 삭제만 가능 (어드민 전용). 삭제는 논리적 삭제(soft delete) — 레코드 보존, 목록 비표시, 이력 조회 가능.

**불변식**:
- INV-05: 매체사 삭제 시 소속 활성 매체(보관됨 제외)가 0건이어야 함 (활성/비활성/미연동 매체가 존재하면 삭제 불가)
- INV-05a: 매체사 삭제 시 소속 활성 사용자가 존재하면 "소속 사용자 N명이 비활성화됩니다. 진행하시겠습니까?" 경고를 표시한다. 확인 시 소속 사용자를 일괄 비활성화한 후 매체사를 삭제한다
- INV-06: 매체사 식별 ID는 시스템 발급, 사용자 변경 불가

### 5.2 매체 상태 FSM

```
[미연동] ──(디바이스 연동 성공)──→ [연동됨(활성)]
    │                                 │        ↑
    │                            (비활성화)  (활성화)
    │                                 ↓        │
    │                            [연동됨(비활성)] ─┘
    │                                 │
    │                            (보관)
    │                                 ↓
    └──────(보관)──────────────→ [보관됨]
                        ↑
              [연동됨(활성)] ─(교체 실패)─→ [미연동]
                                             (재연동 성공 시 → 연동됨(활성))
```

**상태**:
| 상태 | 설명 | 허용 동작 |
|------|------|----------|
| 미연동 | 매체 등록 완료, 디바이스 미연결 | 정보 수정, 디바이스 연동, SSP 설정 |
| 연동됨(활성) | 디바이스 연결, 광고 송출 가능 | 전체 기능 사용, 편성 가능 |
| 연동됨(비활성) | 운영 일시 중지 | 정보 조회, 활성화, 보관 |
| 보관됨 | 철거/계약종료 매체. 기본 목록 비표시, 이력 조회 전용 | 이력 조회만 |

**전이**:
| From → To | 트리거 | 가드 | 효과 |
|-----------|--------|------|------|
| 미연동 → 연동됨(활성) | 디바이스 ID 입력 + 통신 성공 | 플레이어 설치 완료 | 디바이스 연동, 헬스체크 시작 |
| 연동됨(활성) → 연동됨(비활성) | 어드민/매체사/운영대행사가 비활성화 | 권한 보유 | `적용중` 편성표 → `일시정지` 전환, 송출 중지. 비활성화 주체 ≠ 편성 생성자 시 알림 |
| 연동됨(비활성) → 연동됨(활성) | 어드민/매체사/운영대행사가 활성화 | 권한 보유 | `일시정지` 편성표 중 종료일 미경과 → `적용중` 자동 복귀. 종료일 경과 → `종료` 처리 |
| 연동됨(활성) → 연동됨(활성) | 디바이스 교체 성공 | 권한 보유 | 기존 디바이스 ID 해제 → 새 디바이스 ID 연동. 편성표 유지, 재생목록 자동 재전송 |
| 연동됨(활성) → 미연동 | 디바이스 교체 실패 | 기존 디바이스 이미 해제 + 새 디바이스 연동 실패 | 편성표 "일시정지" 전환. 재연동 성공 시 "연동됨(활성)"으로 복귀, 편성표 자동 재개. **기존 해제 + 새 연동 + 편성표 전이를 단일 트랜잭션으로 처리** — 중간 실패 시 전체 롤백 |
| 연동됨(비활성) → 보관됨 | 어드민/매체사가 보관 처리 | 적용중·예약됨 편성표 없음. **비관적 잠금(SELECT FOR UPDATE)으로 가드 검사와 상태 전이를 원자적으로 처리** | 일시정지 편성표가 있으면 확인 모달: "일시정지 편성표 N건이 종료 처리됩니다. 계속하시겠습니까?" → 확인 시 일시정지 편성표 일괄 종료. 디바이스 ID 해제. 기본 목록 비표시. 이력 보존. 소속 매체사의 매체 카운트에서 제외 |
| 미연동 → 보관됨 | 어드민/매체사가 보관 처리 | 편성표 없음 | 기본 목록 비표시. 이력 보존. 소속 매체사의 매체 카운트에서 제외 |

**불변식**:
- INV-07: 디바이스 ID는 시스템 전체에서 고유. 하나의 매체에만 연동 가능
- INV-08: 플레이어 연결 시점은 타 매체와 겹치지 않음 (시스템 자동 분산). 분산 풀 잔여 슬롯이 10% 이하이면 어드민에게 "플레이어 연결 시점 풀 부족" 경고 알림 발송. 풀 고갈 시 매체 등록은 허용하되 연결 시점 할당을 보류하고 어드민에게 수동 조정 요청
- INV-09: 플레이어 연결 시점 수정은 어드민만 가능
- INV-10: 비활성 매체에는 새 편성표를 적용할 수 없음
- INV-10a: 보관됨 매체는 기본 목록에서 비표시. 이력 조회만 가능. 매체사 삭제 시 보관됨 매체는 소속 매체 카운트에서 제외 (INV-05 가드 통과)
- INV-10b: 보관됨 상태는 최종 상태. 보관 해제(재활성화) 불가. 동일 장비를 재사용하려면 새 매체로 등록

### 5.3 비즈니스 규칙

- MED-BR-01: 매체 등록 시 매체 식별 코드(Media Code) 자동 발급. 디바이스 ID는 UC-08에서 현장 장비의 물리 식별자를 입력하여 연동
- MED-BR-02: 플레이어 연결 시점은 등록 시 자동 생성, 타 매체와 분산 배치
- MED-BR-03: SSP 연동은 선택 사항. 1차 네이버, 이후 확장
- MED-BR-04: 유동인구 데이터 연결은 어드민 전용, 선택 사항
- MED-BR-05: 매체 비활성화 시 해당 편성표는 `일시정지` 상태로 전환. 활성화 시 종료일 미경과 편성표는 `적용중` 자동 복귀, 종료일 경과 편성표는 `종료` 처리
