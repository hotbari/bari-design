# 소재 관리 시나리오

## 1. 개요

광고 소재(영상)를 업로드하고, 자동/수동 검수를 거쳐 편성에 사용할 수 있도록 관리한다. 소재는 편성의 최소 재생 단위이다. 허용 포맷은 MP4 단일(H.264 코덱, 30fps 강제)이며, 이미지 소재는 불허한다.

## 4. 상세 시나리오

### UC-13: 소재 업로드

**누가**: 매체사, 운영대행사 또는 영업대행사
**언제**: 광고 소재를 CMS에 등록하여 편성에 사용하고자 할 때
**전제조건**: 로그인 상태. 소속 매체사에 등록된 매체가 1개 이상 존재

#### 시나리오

1. 매체사가 소재 관리 > 업로드 버튼을 클릭한다
2. 파일을 선택한다 (드래그&드롭 또는 파일 탐색기, 복수 파일 동시 선택 가능)
3. 시스템이 파일 기본 검증을 수행한다:
   - 허용 파일 형식: MP4 단일
   - 코덱 검증: H.264만 허용 (업로드 시점 차단)
   - 프레임레이트 검증: 30fps만 허용 (업로드 시점 차단)
   - 최대 파일 크기
4. 메타데이터를 입력한다:
   - 소재명 (필수)
   - 광고주 (선택, 메모 용도. 캠페인 광고주와의 일치 검증 없음)
   - 재생 시간 — 파일에서 자동 추출 (필수)
   - 대상 해상도 (필수)
   - 운영 기간 (시작일/종료일, 필수) — 만료 자동 표시용
   - 대상 매체 선택 (필수) — 매체별 폴더 구분 관리
   - 비고 (선택)
5. 업로드 버튼을 클릭한다
6. 시스템이 파일을 저장하고 UC-14(자동 검수)를 트리거한다
7. 소재 상태가 "검수중"으로 표시된다

#### 이런 경우에는

- **허용되지 않은 파일 형식**: "MP4 파일만 업로드 가능합니다" 안내. 업로드 차단
- **코덱 불일치 (H.264 아님)**: "H.264 코덱의 MP4 파일만 허용됩니다. 현재 파일의 코덱: [코덱명]" 안내. 업로드 차단
- **프레임레이트 불일치 (30fps 아님)**: "30fps MP4 파일만 허용됩니다. 현재 파일: [N]fps" 안내. 업로드 차단
- **파일 크기 초과**: "파일 크기가 제한(XXX MB)을 초과합니다" 안내
- **동일 파일명 업로드**: 시스템이 자동으로 파일명 뒤에 넘버링 부여 (예: 광고A_2.mp4)
- **복수 파일 동시 업로드 (일괄 업로드)**: 필수 기능. 각 파일별 메타데이터 개별 입력 또는 일괄 적용. 말일 편성 집중 시 대량 업로드 대응
- **영업대행사의 접근 범위 밖 소재 업로드 시도**: 해당 매체사의 소재 관리 메뉴 자체가 표시되지 않음

#### 결과

- 소재 레코드 생성: 상태 "검수중"
- 파일 스토리지에 원본 저장
- UC-14 자동 검수 트리거

### UC-14: 소재 자동 검수

**누가**: 시스템 (자동)
**언제**: UC-13에서 소재 업로드 완료 직후
**전제조건**: 소재가 "검수중" 상태

#### 시나리오

1. 시스템이 업로드된 소재를 자동 검수한다:
   - 해상도 검증: 대상 해상도와 실제 파일 해상도 일치 여부
   - 용량 검증: 허용 범위 내 여부
   - 코덱 검증: H.264 코덱 여부
   - 프레임레이트 검증: 30fps 여부
   - 재생 시간 검증: 메타데이터와 실제 재생 시간 일치
   - 파일 무결성: 손상 여부
   - **재생·캡처 자동 검증**: 실제 디코딩 후 프레임 캡처로 정상 재생 여부 확인 (인코딩 오류로 화면 하얗게 출력되는 사례 방지)
2. 모든 항목 통과 시 소재 상태가 "검수완료"로 변경된다
3. 썸네일 자동 생성 (소재 목록 표시용)
4. 소재 등록자에게 "검수 완료" 알림 발송

#### 이런 경우에는

- **해상도 불일치**: 소재 상태 "검수실패". 실패 사유: "등록 해상도(1920x1080)와 실제 해상도(1280x720)가 다릅니다". 등록자에게 "검수 실패" 알림 발송
- **파일 손상**: 소재 상태 "검수실패". 실패 사유: "파일이 손상되었습니다. 다시 업로드해주세요"
- **자동 검수로 판단 불가한 항목 존재**: 소재 상태 "수동검수대기". 검수 권한자에게 "사용자 확인 필요" 알림 발송, 소재 등록자에게 "소재가 수동 검수 대기 중입니다. 검수 담당자가 확인 후 처리 예정입니다" 정보성 알림 발송 → UC-15로 이동
- **자동 검수 타임아웃**: 자동 검수가 10분 이내에 완료되지 않으면 "검수실패" 상태로 전환. 실패 사유: "검수 시간 초과. 소재를 다시 업로드해주세요". 소재 등록자에게 알림 발송

#### 결과

- 검수 통과: 소재 상태 "검수중" → "검수완료". 편성에 사용 가능
- 검수 실패: 소재 상태 "검수중" → "검수실패". 재업로드 필요
- 수동 검수 필요: 소재 상태 "검수중" → "수동검수대기"

### UC-15: 소재 수동 검수 확인

**누가**: 어드민 또는 매체사
**언제**: UC-14에서 "수동검수대기" 상태가 된 소재를 확인할 때
**전제조건**: 소재가 "수동검수대기" 상태. 검수 권한 보유

#### 시나리오

1. 어드민 또는 매체사가 소재 관리 > 검수 대기 목록을 확인한다
2. 대상 소재를 선택하여 미리보기로 확인한다
3. 자동 검수 결과 리포트를 확인한다 (통과 항목, 확인 필요 항목)
4. "검수 통과" 또는 "검수 실패" 버튼을 클릭한다
5. 실패 시 사유를 입력한다

#### 이런 경우에는

- **수동검수대기 소재가 장기간 방치**: 일정 기간(예: 7일) 경과 시 어드민에게 리마인더 알림
- **미리보기 불가 (인코딩 문제)**: "미리보기를 지원하지 않는 형식입니다. 파일을 다운로드하여 확인해주세요" 안내 + 다운로드 버튼
- **수동검수대기 상태에서 파일 교체 시도**: "현재 검수 진행 중입니다. 검수 완료 후 교체해주세요" 안내. 교체 차단. 검수 완료 또는 검수 실패 후 교체 가능
- **검수 실패 시 사유 미입력**: "검수 실패" 선택 시 사유 입력 필수. 최소 10자 이상 입력해야 한다. 자동 검수 실패 시에도 시스템이 구체적 사유를 자동 생성한다 (예: "해상도 불일치: 입력 1920×1080, 실제 1280×720")
- **수동검수대기 강제 타임아웃**: 수동검수대기 상태가 14일을 초과하면 시스템이 자동으로 "검수실패"로 전환한다. 실패 사유: "장기 미응답으로 검수 거절됨. 소재를 다시 업로드해주세요". 소재 등록자와 어드민에게 알림 발송

#### 결과

- 검수 통과: 소재 상태 "수동검수대기" → "검수완료". 등록자에게 "검수 완료" 알림
- 검수 실패: 소재 상태 "수동검수대기" → "검수실패". 등록자에게 "검수 실패" 알림 + 실패 사유

### UC-16: 소재 상태 관리 (교체/삭제)

**누가**: 매체사, 운영대행사 또는 영업대행사
**언제**: 기존 소재를 교체하거나 삭제할 때
**전제조건**: 소재가 존재. 해당 소재에 대한 관리 권한 보유

#### 시나리오 A — 소재 교체

1. 매체사가 소재 관리 > 소재 상세에서 "교체" 버튼을 클릭한다
2. 새 파일을 선택한다
3. 시스템이 기존 메타데이터를 유지한 채 파일만 교체한다
4. UC-14 자동 검수가 다시 트리거된다
5. 소재 상태가 "검수중"으로 변경된다

#### 시나리오 B — 소재 삭제

1. 매체사가 소재 관리 > 소재 상세에서 "삭제" 버튼을 클릭한다
2. 시스템이 확인 모달을 표시한다

#### 이런 경우에는

- **편성에 사용 중인 소재 교체**: "이 소재는 현재 N건의 재생목록에 사용 중입니다. 교체하면 검수 완료 후 자동 반영됩니다. 계속하시겠습니까?" 확인
- **편성에 사용 중인 소재 삭제**: "이 소재는 현재 N건의 재생목록에 사용 중입니다. 삭제하면 해당 구좌가 '삭제됨' 상태로 표시됩니다. 계속하시겠습니까?" 경고. 삭제 후 편성 관리자에게 알림 발송
- **검수 실패 소재 재업로드**: 소재 상세에서 "재업로드" 버튼 → UC-13과 동일 플로우 (기존 메타데이터 유지)
- **검수중 소재 삭제**: 검수 프로세스 중단 후 삭제
- **수동검수대기 상태에서 파일 교체 시도**: "현재 검수 진행 중입니다. 검수 완료 후 교체해주세요" 안내. 교체 차단
- **소재 교체 중 편성 참조 레이스 컨디션**: 소재 교체(파일 업로드 → 재검수) 시작 시, 해당 소재를 참조하는 재생목록의 구좌 상태를 "교체중"으로 표시. "교체중" 구좌는 기존 파일로 계속 재생 (신규 파일은 검수 완료 전까지 송출하지 않음). 재검수 완료 시 신규 파일로 자동 전환, 구좌 상태 "정상" 복귀. 재검수 실패 시 기존 파일 유지, 등록자에게 알림

#### 결과

- 교체: 파일만 변경, 메타데이터 유지, 재검수 시작
- 삭제: 소재 레코드 삭제. 재생목록 내 해당 구좌는 "삭제됨" 상태로 전환 (구좌 구조 보존, 재생 시 스킵). 편성 관리자에게 알림. 파일 스토리지에서 삭제

### 소재 운영 상태

소재에는 검수축 상태와 별개로, 운영 기간 기반의 **운영 상태**가 표시된다.

| 운영 상태 | 조건 | 설명 |
|----------|------|------|
| 예정 | 현재일 < 운영 시작일 | 운영 기간 시작 전 |
| 진행중 | 운영 시작일 ≤ 현재일 ≤ 운영 종료일 | 운영 기간 내 |
| 만료 | 현재일 > 운영 종료일 | 운영 기간 종료 |

- 소재 목록에 운영 상태 필터(진행중/만료/예정) 제공
- 소재 목록에 **썸네일 보기 모드** 제공 (리스트 뷰 / 썸네일 뷰 전환)
- 만료 소재는 목록에서 시각적 구분 표시 (흐리게 처리)

## 5. 도메인 규칙

### 5.1 소재 상태 FSM (검수축)

```
[검수중] ──(자동 검수 통과)──→ [검수완료]
   │                              ↑
   ├──(자동 검수 실패)──→ [검수실패] ──(재업로드)──→ [검수중]
   │
   └──(수동 검수 필요)──→ [수동검수대기]
                              │
                              ├──(통과)──→ [검수완료]
                              └──(실패)──→ [검수실패]
```

**상태**:
| 상태 | 설명 | 허용 동작 |
|------|------|----------|
| 검수중 | 자동 검수 진행 중 | 삭제 |
| 검수완료 | 검수 통과, 편성 사용 가능 | 편성 연결, 교체, 삭제 |
| 검수실패 | 검수 미통과 | 재업로드, 삭제 |
| 수동검수대기 | 수동 확인 필요 | 수동 검수, 삭제 |

**전이**:
| From → To | 트리거 | 가드 | 효과 |
|-----------|--------|------|------|
| 검수중 → 검수완료 | 자동 검수 전 항목 통과 | — | 알림: "검수 완료" |
| 검수중 → 검수실패 | 자동 검수 항목 실패 | — | 알림: "검수 실패" + 사유 |
| 검수중 → 수동검수대기 | 자동 판단 불가 항목 존재 | — | 알림: 검수 권한자에게 "사용자 확인 필요", 소재 등록자에게 "수동 검수 대기 중" 정보성 알림 |
| 수동검수대기 → 검수완료 | 검수 권한자가 통과 처리 | 검수 권한 보유 | 알림: "검수 완료" |

### 5.2 비즈니스 규칙

- MAT-BR-01: 소재 업로드 시 자동 검수가 기본 (확정 의사결정: 자동 검수 강화)
- MAT-BR-02: 수동 검수는 자동으로 판단 불가한 경우에만 발생하는 예외 케이스
- MAT-BR-03: 소재 교체 시 메타데이터 유지, 파일만 교체, 재검수 필수
- MAT-BR-04: 편성에 사용 중인 소재 삭제 시 재생목록 내 구좌는 "삭제됨" 상태로 보존 (구좌 순서 유지, 재생 시 스킵). 편성 관리자에게 알림 발송. UI에서 빨간 점선 + "소재 삭제됨 — 교체 필요" 표시
- MAT-BR-05a: 소재의 광고주 필드는 메모 용도. 캠페인 광고주와의 일치 검증 없음
- MAT-BR-05b: 영업대행사의 소재 목록 조회는 기본적으로 본인이 등록한 소재만 표시. 동일 광고주를 담당하는 영업대행사 간에는 광고주 단위 공유 설정 가능
- MAT-BR-06: 허용 파일 포맷은 MP4 단일 (H.264 코덱, 30fps). 업로드 시점에 강제 검증. 이미지 소재 불허
- MAT-BR-07: 소재는 대상 매체별로 폴더 구분 관리 (동일 광고도 매체 해상도에 따라 별도 파일)
