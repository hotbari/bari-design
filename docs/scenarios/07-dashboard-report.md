# 대시보드/리포트 시나리오

## 1. 개요

CMS의 전체 운영 현황을 대시보드로 시각화하고, 정기/수동 리포트를 생성·발송한다. 매체 상태, 캠페인 현황, 편성 상태, 알림 요약 등을 한눈에 파악할 수 있다.

## 4. 상세 시나리오

### UC-29: 대시보드 현황 조회

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: CMS 로그인 직후 또는 운영 현황을 확인할 때
**전제조건**: 로그인 상태

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 CMS에 로그인하면 대시보드가 기본 화면으로 표시된다
2. 대시보드에 다음 위젯이 표시된다:
   - **매체 상태 요약**: 전체/온라인/오프라인/비활성 매체 수
   - **캠페인 현황**: 초안/집행중/완료/취소 건수
   - **편성 상태**: 예약됨/적용중/일시정지/종료 건수
   - **최근 알림**: 안 읽은 알림 상위 5건
   - **매체 헬스체크 이상**: 이상 감지 매체 목록
   - **매체 위치 지도**: 위경도 기반 매체 위치 시각화 (지도 위젯)
   - **편성 동기화 상태**: 동기화 미완료 편성표 N건 (어드민 전용)
   - **배치 실행 상태**: 최종 배치 실행 시각 + 비정상 캠페인(시작일 경과+초안) N건 (어드민 전용)
   - **Slack 연동 상태**: 각 Webhook의 마지막 성공 발송 시각 (어드민 전용)
   - **영업대행사 소재 현황** (영업대행사 로그인 시): 상태별 소재 건수, 만료 임박 소재, 검수실패 소재
3. 각 위젯을 클릭하면 해당 상세 화면으로 이동한다
4. 기간 필터를 적용하여 특정 기간의 현황을 확인할 수 있다

#### 이런 경우에는

- **매체사 사용자**: 본인 매체 관련 데이터만 표시
- **운영대행사**: 부여받은 매체 관련 데이터만 표시
- **데이터 없음 (신규 매체사)**: 각 위젯에 "아직 데이터가 없습니다" 안내

#### 결과

- 권한 범위 내 운영 현황 시각화

### UC-30: 리포트 생성 및 예약 발송

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 운영 실적 리포트를 생성하거나 정기 발송을 예약할 때
**전제조건**: 로그인 상태. 리포트 대상 데이터 존재

#### 시나리오 A — 수동 리포트 생성

1. 어드민, 매체사 또는 운영대행사가 리포트 > 새 리포트 버튼을 클릭한다
2. 리포트 유형을 선택한다:
   - 매체 운영 리포트 (매체별 온라인 시간, 재생 횟수 등)
   - 캠페인 실적 리포트 (캠페인별 집행 현황)
   - 편성 리포트 (편성별 재생 통계)
3. 대상 기간을 설정한다 (시작일~종료일)
4. 대상 매체를 선택한다 (전체 또는 특정 매체)
5. 생성 버튼을 클릭한다
6. 시스템이 리포트를 생성한다
7. 생성 완료 시 리포트 목록에 표시되고, 다운로드 가능

#### 시나리오 B — 예약 발송 설정

1. 리포트 생성 화면에서 "예약 발송" 토글을 ON으로 설정한다
2. 발송 주기를 선택한다: 매일 / 매주 / 매월
3. 수신자 이메일을 입력한다 (복수 가능)
4. 저장 버튼을 클릭한다
5. 시스템이 설정된 주기마다 리포트를 자동 생성·발송한다

#### 이런 경우에는

- **대상 기간에 데이터 없음**: "해당 기간에 데이터가 없습니다" 안내. 빈 리포트 생성 가능 (데이터 없음 표시)
- **예약 발송 메일 발송 실패**: 리포트 설정자에게 "리포트 발송 실패" 알림 (UC-26 연계)
- **예약 발송 수정/삭제**: 리포트 > 예약 목록에서 수정/삭제 가능
- **운영대행사가 권한 범위 밖 매체 데이터 리포트 생성 시도**: 해당 매체가 선택 목록에 표시되지 않음

#### 결과

- 수동: 리포트 파일 생성 (다운로드 가능)
- 예약: 주기적 자동 생성 + 이메일 발송

### UC-31: 리포트 조회/다운로드

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 기존에 생성된 리포트를 확인하거나 다운로드할 때
**전제조건**: 로그인 상태. 리포트가 1건 이상 존재

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 리포트 > 리포트 목록을 클릭한다
2. 생성된 리포트 목록이 표시된다 (권한 범위 내)
3. 필터를 적용한다:
   - 리포트 유형
   - 생성 기간
   - 대상 매체
4. 리포트를 선택하면 미리보기가 표시된다
5. "다운로드" 버튼으로 파일을 다운로드한다

#### 이런 경우에는

- **리포트 목록 없음**: "생성된 리포트가 없습니다" 안내
- **리포트 파일 손상**: "리포트를 열 수 없습니다. 다시 생성해주세요" 안내
- **대량 리포트 다운로드**: 일괄 다운로드 지원 (ZIP 압축)

#### 결과

- 리포트 미리보기 및 파일 다운로드

## 6. 미결 사항

- 리포트 파일 형식 미정의 (PDF? Excel? CSV?)
- 대시보드 위젯 커스터마이징(순서 변경, 표시/숨김) 가능 여부 미정의
- 리포트 보존 기간 미정의 (무제한? 일정 기간 후 자동 삭제?)
- 유동인구 데이터가 연결된 매체의 경우 리포트에 유동인구 통계 포함 여부 미정의
- 리포트 내 차트/그래프 유형 미정의
- 실시간 대시보드 갱신 주기 미정의 (웹소켓? 폴링? 수동 새로고침?)
- 매체사별 월간 리포트 (RFP 산출물: "매체사 세일즈 관련 월간 리포트") 자동 생성 여부 미정의
- 예약 발송 "매월" 옵션의 기준일 미정의 (매월 1일? 설정일 기준?)
- 대량 매체 리포트 생성 시 비동기 처리 + 완료 알림 패턴 필요 여부 미정의
- 리포트 DB 성능 최적화 필요 (레포트 로딩 비정상적 느림 현상 — 회의 3 확인)
- **[인터뷰 도출]** 대시보드 위젯별 최종 갱신 타임스탬프 표시 필요 — 매체 상태는 실시간, 나머지는 주기적 갱신. 갱신 주기 미정의
