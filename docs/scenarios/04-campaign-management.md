# 캠페인 관리 시나리오

## 1. 개요

캠페인은 "누가 얼마를 내고 어디에 광고하는지"를 정의하는 비즈니스 단위이다. 광고주, 대상 매체, 집행 기간, 예산 정보를 포함한다. 캠페인은 편성표를 직접 제어하지 않으며, 각각 독립적으로 관리된다. **승인/반려 워크플로우 없음** (확정 의사결정).

## 4. 상세 시나리오

### UC-17: 캠페인 등록 (3단계 스텝)

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 새로운 광고 계약을 CMS에 등록할 때
**전제조건**: 로그인 상태. 대상 매체가 UC-07로 등록 완료

#### 시나리오

1. 매체사가 캠페인 관리 > 등록 버튼을 클릭한다
2. 상단에 단계 표시: **1.기본 정보 → 2.매체 및 기간 → 3.예산 및 확인**

**Step 1: 기본 정보**
3. 캠페인명을 입력한다 (필수)
4. 광고주를 선택한다 (드롭다운 단일 선택, 필수)
   - 옆에 [신규 등록] 버튼 → 광고주 등록 모달 → 등록 완료 시 자동 선택
5. 광고대행사를 입력한다 (텍스트 직접 입력, 선택)
6. 캠페인 유형을 선택한다 (필수, 단일): 직접판매 / 자사광고 / 필러 / 네이버
7. 광고 유형을 선택한다 (단일)
8. [다음] 버튼을 클릭한다

**Step 2: 매체 및 기간**
9. 대상 매체를 선택한다 (개별 매체 또는 매체 그룹, 복수 선택, 최소 1개 필수)
10. 집행 기간을 설정한다 (시작일·종료일, 필수)
11. [다음] 버튼을 클릭한다

**Step 3: 예산 및 확인**
12. 과금 모델을 선택한다 (단일): 정액제 / 일단가 / 입력안함
13. 과금 모델 선택 시 집행 금액을 입력한다 ("입력안함" 선택 시 미표시)
14. 전체 입력 내용을 확인한다
15. [등록] 버튼을 클릭한다
16. 시스템이 캠페인을 "초안" 상태로 생성한다

#### 이런 경우에는

- **필수 정보 미입력 시 [다음] 클릭**: 버튼 비활성화. 미입력 필드 하이라이트
- **시작일 > 종료일**: "시작일이 종료일보다 앞서야 합니다" 안내
- **시작일이 과거 날짜**: "시작일은 오늘 이후여야 합니다" 안내
- **광고주 신규 등록 모달에서 취소**: 모달 닫힘, 광고주 미선택 상태 유지
- **운영대행사가 권한 범위 밖 매체 선택 시도**: 해당 매체가 선택 목록에 표시되지 않음
- **집행중 캠페인에서 대상 매체 제거**: 해당 매체에 캠페인 FK로 연결된 편성표가 있으면 경고: "이 매체에 연결된 편성표 N건이 있습니다. 매체를 제거하면 연결 편성표의 캠페인 FK가 해제됩니다 (편성표 자체는 유지). 계속하시겠습니까?" 확인 시 해당 매체의 편성표에서 캠페인 FK만 해제 (편성표 종료 아님)
- **[이전] 버튼 클릭**: 이전 단계로 이동. 입력 내용 유지
- **등록 중 브라우저 이탈**: 입력 내용 미저장. "작성 중인 내용이 있습니다. 이탈하시겠습니까?" 확인
- **동일 매체 + 겹치는 기간에 기존 캠페인 존재 시 (Step 3 저장)**: 동일 매체에 "초안" 또는 "집행중" 상태의 캠페인이 겹치는 기간에 존재하면 경고 표시 — "동일 매체에 겹치는 기간의 캠페인이 N건 존재합니다. 편성 시 충돌이 발생할 수 있습니다." 경고 후 저장은 허용 (편성 충돌 해소는 편성 도메인 UC-24에서 처리)

#### 결과

- 캠페인 레코드 생성: 상태 "초안"
- 시작일 도래 시 자동으로 UC-18(집행 시작) 트리거

### UC-17a: 캠페인 수정

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 등록된 캠페인의 정보를 변경할 때
**전제조건**: 캠페인이 "초안" 또는 "집행중" 상태. 해당 캠페인에 대한 수정 권한 보유

#### 시나리오

1. 매체사가 캠페인 관리 > 캠페인 상세에서 "수정" 버튼을 클릭한다
2. 캠페인 상태에 따라 수정 가능 필드가 다르다:

**"초안" 상태 — 전체 수정 가능**:
3. UC-17과 동일한 3단계 스텝으로 모든 필드 수정 가능

**"집행중" 상태 — 제한적 수정**:
3. 수정 가능 필드만 활성화된다:
   - 대상 매체 추가/제거 (CAM-BR-06 가드 적용)
   - 종료일 변경
   - 집행 금액 변경
4. 수정 불가 필드는 비활성화 표시 + "집행중에는 변경할 수 없습니다" 툴팁:
   - 캠페인명, 광고주, 캠페인 유형, 시작일
5. 저장 버튼을 클릭한다

#### 이런 경우에는

- **종료일을 현재보다 앞으로 단축**: "연결 편성표 중 종료일이 변경된 캠페인 종료일을 초과하는 건이 N건 있습니다. 편성표 종료일도 동기화하시겠습니까?" 확인. 동기화 선택 시 해당 편성표 종료일을 캠페인 종료일로 변경
- **집행중 캠페인에서 마지막 매체 제거 시도**: "이 캠페인의 유일한 대상 매체입니다. 제거하려면 캠페인을 취소해주세요" 안내. 제거 차단 (INV-15 + CAM-BR-06)
- **완료/취소 상태 캠페인 수정 시도**: "수정" 버튼이 표시되지 않음 (INV-14)

#### 결과

- 캠페인 정보 업데이트
- 종료일 단축 시 연결 편성표 동기화 (사용자 선택)

### UC-18: 캠페인 집행 시작

**누가**: 시스템 (자동)
**언제**: 캠페인 시작일이 도래했을 때
**전제조건**: 캠페인이 "초안" 상태. 시작일 = 오늘

#### 시나리오

1. 시스템이 매일 정해진 시간에 시작일이 도래한 캠페인을 조회한다
2. 해당 캠페인의 상태를 "초안" → "집행중"으로 변경한다
3. 캠페인 상태 변경이 목록에 반영된다 (기본 알림 없음 — 캠페인은 비즈니스 단위, 실제 송출은 편성이 담당)

#### 이런 경우에는

- **시작일 도래 시점에 대상 매체가 비활성 상태**: 캠페인은 "집행중"으로 전환되지만, 해당 매체에서는 실제 광고 송출 불가. **예외 알림 발송**: "캠페인 [캠페인명]의 시작일이나 대상 매체 중 비활성 매체가 있습니다" (캠페인 등록자에게)
- **시작일이 이미 지난 캠페인이 "초안" 상태로 남아 있음**: 시스템이 감지하여 "집행중"으로 자동 전환 (소급 처리)

#### 결과

- 캠페인 상태: "초안" → "집행중"
- 기본 알림 없음. 예외 조건(대상 매체 비활성 등) 발생 시에만 알림 발송

### UC-19: 캠페인 완료/취소

**누가**: 시스템 (자동 완료) 또는 어드민, 매체사, 운영대행사 (수동 취소)
**언제**: 캠페인 종료일이 도래하거나, 사용자가 캠페인을 취소할 때
**전제조건**: 캠페인이 "초안" 또는 "집행중" 상태

#### 시나리오 A — 자동 완료

1. 시스템이 매일 정해진 시간에 종료일이 경과한 캠페인을 조회한다
2. 해당 캠페인의 상태를 "집행중" → "완료"로 변경한다
3. 캠페인 상태 변경이 목록에 반영된다
4. 연결된 편성표가 아직 적용중인 경우: 캠페인 등록자에게 "캠페인 [캠페인명] 완료 — 연결 편성표 N건이 아직 적용중입니다. 편성표 종료 여부를 확인해주세요" 알림 발송. 편성표 자체는 자동 종료하지 않음 (편성표는 자체 종료일까지 독립 운영)

#### 시나리오 B — 수동 취소

1. 어드민, 매체사 또는 운영대행사가 캠페인 상세에서 "취소" 버튼을 클릭한다
2. 취소 사유를 입력한다 (필수)
3. 시스템이 확인 모달을 표시한다: "이 캠페인을 취소하시겠습니까? 연결된 편성표가 자동 종료됩니다."
4. 확인 버튼을 누르면 캠페인 상태가 "취소"로 변경된다
5. 캠페인 취소 시, 해당 캠페인에 연결된 예약됨/적용중 상태의 편성표를 자동 종료한다. 편성 담당자에게 '캠페인 취소로 편성표 N건 종료됨' 알림 발송. (캠페인-편성표 연결은 선택 필드 — 미연결 편성표는 영향 없음)

#### 이런 경우에는

- **집행중 캠페인 취소 시**: "집행 중인 캠페인입니다. 취소 시 연결된 편성표가 자동 종료됩니다." 경고
- **완료된 캠페인 취소 시도**: "취소" 버튼이 표시되지 않음 (완료는 최종 상태)
- **이미 취소된 캠페인 재활성화**: 불가. 새 캠페인으로 등록 필요
- **매체 비활성화 시 "집행중" 캠페인 존재**: 해당 매체를 대상으로 하는 "집행중" 캠페인의 담당자에게 알림 발송. 대상 매체 전부 비활성 → "모든 대상 매체 비활성 — 실제 송출 불가" 경고 알림. 대상 매체 일부 비활성 → "N개 매체 비활성 — 해당 매체 송출 중단" 정보 알림. 캠페인 상태는 "집행중" 유지 (편성표 도메인에서 일시정지 처리)

#### 결과

- 자동 완료: 상태 "집행중" → "완료"
- 수동 취소: 상태 "초안"/"집행중" → "취소". 취소 사유 기록

### UC-20: 캠페인 목록 조회 및 필터링

**누가**: 어드민, 매체사 또는 운영대행사
**언제**: 등록된 캠페인 현황을 확인할 때
**전제조건**: 로그인 상태

#### 시나리오

1. 어드민, 매체사 또는 운영대행사가 캠페인 관리 메뉴를 클릭한다
2. 캠페인 목록이 표시된다 (권한 범위 내)
3. 필터를 적용한다:
   - 상태: 초안 / 집행중 / 완료 / 취소
   - 캠페인 유형: 직접판매 / 자사광고 / 필러 / 네이버
   - 집행 기간: 시작일~종료일 범위
   - 광고주
   - 대상 매체
4. 검색어로 캠페인명을 검색한다
5. 정렬 기준을 변경한다 (등록일, 시작일, 종료일, 캠페인명)

#### 이런 경우에는

- **매체사 사용자**: 본인 매체에 연결된 캠페인만 표시
- **운영대행사**: 부여받은 매체에 연결된 캠페인만 표시
- **검색 결과 없음**: "조건에 맞는 캠페인이 없습니다" 안내
- **대량 데이터**: 페이지네이션 적용 (기본 20건)

#### 결과

- 권한 범위 내 캠페인 목록 표시
- 필터/검색/정렬 적용

## 5. 도메인 규칙

### 5.1 캠페인 상태 FSM

```
[초안] ──(시작일 도래)──→ [집행중] ──(종료일 경과)──→ [완료]
  │                         │
  └──(수동 취소)──→ [취소] ←──(수동 취소)──┘
```

**상태**:
| 상태 | 설명 | 허용 동작 |
|------|------|----------|
| 초안 | 등록 완료, 집행 전 | 수정, 취소 |
| 집행중 | 시작일 경과, 광고 운영 중 | 수정(대상 매체 추가/제거, 종료일 변경, 금액 변경), 취소 |
| 완료 | 종료일 경과, 최종 상태 | 조회만 |
| 취소 | 사용자가 취소, 최종 상태 | 조회만 |

**전이**:
| From → To | 트리거 | 가드 | 효과 |
|-----------|--------|------|------|
| 초안 → 집행중 | 시작일 도래 (시스템 자동) | — | 상태 변경. 대상 매체 비활성 시 경고 포함 |
| 집행중 → 완료 | 종료일 경과 (시스템 자동) | — | 상태 변경 |
| 초안 → 취소 | 사용자가 취소 | 취소 사유 입력 | 상태 변경, 사유 기록 |
| 집행중 → 취소 | 사용자가 취소 | 취소 사유 입력 | 즉시 집행 중단, 사유 기록, 연결 편성표 연쇄 종료. **캠페인 상태 변경 + 편성표 종료 + 알림 발송을 단일 트랜잭션으로 처리** |

**불변식**:
- INV-14: 완료/취소 상태의 캠페인은 수정 불가
- INV-15: 캠페인은 최소 1개의 대상 매체를 가져야 함
- INV-16: 캠페인 시작일 < 종료일
- INV-17: 승인/반려 워크플로우 없음 (등록 즉시 초안, 시작일에 자동 집행)

### 5.2 비즈니스 규칙

- CAM-BR-01: 캠페인은 편성표를 직접 제어하지 않음. 각각 독립 관리
- CAM-BR-02: 캠페인 유형 및 비즈니스 의미:
  - **직접판매**: 외부 광고주와의 유상 계약. 과금 모델 적용 대상
  - **자사광고**: 매체사 자체 홍보. 과금 없음 ("입력안함")
  - **필러**: 빈 구좌 채움용 공익광고/하우스애드. 잔여 구좌 조회(UC-36) 시 공익광고 의무 비율(20%) 산출에 포함
  - **네이버**: 네이버 SSP 연동(UC-09) 매체 대상. 편성 시 네이버 Ad Server 연동 구좌(UC-23) 사용 전제
- CAM-BR-03: 과금 모델: 정액제 / 일단가 / 입력안함
- CAM-BR-04: 영업대행사는 캠페인 관리 기능에 접근 불가
- CAM-BR-05: 캠페인 상태 자동 전환은 매일 00:00(KST)에 일 단위로 실행한다. 배치 실패 시 어드민에게 "캠페인 상태 전환 배치 실패" 긴급 알림을 발송한다. 다음 배치에서 미전환 캠페인을 소급 처리한다 (UC-18 소급 처리와 동일)
- CAM-BR-06: 집행중 캠페인에서 대상 매체를 제거하면, 해당 매체의 편성표에서 캠페인 FK만 해제한다. 편성표 자체는 종료하지 않고 독립 운영을 유지한다. **단, 마지막 남은 매체는 제거 불가** (INV-15 가드: "이 캠페인의 유일한 대상 매체입니다. 제거하려면 캠페인을 취소해주세요")
- CAM-BR-07: 캠페인 유형별 제약:
  - **직접판매**: 과금 모델 필수 (정액제 또는 일단가). "입력안함" 선택 불가
  - **자사광고**: 과금 모델 "입력안함" 고정. 금액 입력 불가
  - **필러**: 과금 모델 "입력안함" 고정. 잔여 구좌 조회(UC-36)에서 공익광고 의무 비율 산출에 포함
  - **네이버**: 대상 매체가 네이버 SSP 연동(UC-09) 완료 상태여야 함. 미연동 매체 선택 시 "이 매체는 네이버 SSP 연동이 필요합니다" 경고

## 6. 미결 사항

- ~~집행중 캠페인의 수정 가능 범위 미정의~~ → 대상 매체 추가/제거, 종료일 변경, 금액 변경 허용으로 확정 (I-1). 캠페인명, 광고주, 캠페인 유형, 시작일은 변경 불가. 종료일 단축 시 연결 편성표 적용 기간 초과 경고 + 편성표 종료일 동기화 여부 확인
- ~~캠페인과 편성표의 연관 관계 표시 방법 미정의~~ → 편성표에 캠페인 FK 선택 필드 추가로 확정 (C-10 해소)
- ~~광고주 엔티티의 상세 속성 및 관리 시나리오 미정의~~ → 광고주 필수 속성 확정: 회사명, 사업자등록번호(유니크 키, 중복 등록 방지), 담당자명, 연락처 (I-4). 광고주별 캠페인 이력 조회 기능 필요. 별도 관리 화면은 MVP 이후 검토
- 캠페인 복제(기존 캠페인 기반 신규 생성) 기능 필요 여부 미정의
- ~~시작일 도래 시 자동 전환의 실행 시점 (매일 00:00? 특정 시간?) 미정의~~ → CAM-BR-05로 확정
- 취소된 캠페인의 실적 데이터 보존 방안 미정의
- 계약 관리 기능 폐지 범위 확정 필요: 공익광고/하우스애드/필러 등 필수 카테고리 분류만 잔존 (회의 2 합의). 현재 캠페인 유형(직접판매/자사광고/필러/네이버)이 이를 이미 커버하는지 확인 필요
